
# Java 异常处理

程序运行中遇到的"错误"叫**异常（Exception）**。Java 提供了一套机制来**捕获并处理**这些异常，让程序不会崩溃。

## 一、异常的体系

```
Throwable
├── Error          → 严重错误，程序无法处理（如内存溢出）
└── Exception      → 程序可以处理的异常
    ├── RuntimeException   → 运行时异常（非受检）
    │   ├── NullPointerException        空指针
    │   ├── ArrayIndexOutOfBoundsException  数组越界
    │   ├── ArithmeticException         算术异常（如除以 0）
    │   └── ClassCastException          类型转换异常
    └── IOException 等            → 受检异常（必须处理）
```

**两大类异常：**

| 类型        | 说明                       | 例子                          |
| --------- | ------------------------ | --------------------------- |
| **受检异常**  | 编译时强制处理，否则编译不通过          | `IOException`、`SQLException` |
| **非受检异常**（运行时异常） | 编译时不强制处理，运行时才暴露   | `NullPointerException` 等    |

## 二、try-catch 捕获异常

```java
try {
    int[] arr = {1, 2, 3};
    System.out.println(arr[10]);   // 数组越界，会抛异常
} catch (ArrayIndexOutOfBoundsException e) {
    System.out.println("数组越界啦：" + e.getMessage());
}

System.out.println("程序继续运行");   // 不会因为上面的异常而中断
```

### 多个 catch

```java
try {
    // 一些可能抛多种异常的代码
    String s = null;
    s.length();
} catch (NullPointerException e) {
    System.out.println("空指针");
} catch (Exception e) {
    System.out.println("其他异常：" + e.getMessage());
}
```

> ⚠️ 注意：catch 顺序要从**子类到父类**，否则编译报错。

### 多重捕获（Java 7+）

```java
try {
    // ...
} catch (IOException | SQLException e) {
    System.out.println("IO 或 SQL 异常");
}
```

## 三、finally

无论是否发生异常，**finally 块都会执行**。常用于关闭资源（文件、数据库连接等）。

```java
try {
    System.out.println("尝试执行");
    int x = 10 / 0;   // 抛异常
} catch (ArithmeticException e) {
    System.out.println("捕获异常");
} finally {
    System.out.println("无论如何都会执行");
}

// 输出：
// 尝试执行
// 捕获异常
// 无论如何都会执行
```

## 四、try-with-resources（Java 7+）

自动关闭实现了 `AutoCloseable` 接口的资源，比 `finally` 优雅。

```java
// 旧写法
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try { br.close(); } catch (IOException e) {}
    }
}

// 新写法（推荐）
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
}
// 自动调用 br.close()
```

## 五、throw 与 throws

### throw —— 主动抛出异常

```java
public void setAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("年龄不能是负数");
    }
    this.age = age;
}
```

### throws —— 声明方法可能抛出的异常

```java
public void readFile(String path) throws IOException {
    // 如果方法内部不处理 IOException，必须用 throws 声明
    BufferedReader br = new BufferedReader(new FileReader(path));
    // ...
}

// 调用者必须处理
try {
    readFile("test.txt");
} catch (IOException e) {
    e.printStackTrace();
}
```

**对比：**

| 关键字    | 位置   | 作用              |
| ------ | ---- | --------------- |
| throw  | 方法体内 | 实际抛出一个异常对象      |
| throws | 方法签名 | 声明这个方法可能抛出哪些异常  |

## 六、自定义异常

继承 `Exception`（受检）或 `RuntimeException`（非受检）即可。

```java
// 自定义异常类
public class AgeInvalidException extends RuntimeException {
    public AgeInvalidException(String message) {
        super(message);
    }
}

// 使用
public class Person {
    private int age;

    public void setAge(int age) {
        if (age < 0 || age > 150) {
            throw new AgeInvalidException("年龄不合法：" + age);
        }
        this.age = age;
    }
}
```

## 七、异常处理的最佳实践

✅ **该做的：**
- 捕获**具体的异常**，不要用 `catch (Exception e)` 一把梭
- 异常信息要**清晰具体**，方便排查问题
- 用 `try-with-resources` 自动管理资源
- 自定义异常表达业务含义（如 `UserNotFoundException`）

❌ **不该做的：**
- 不要"吞掉"异常（catch 后什么都不做）
- 不要用异常做正常的流程控制
- 不要在 `finally` 里 `return`，会覆盖 try 中的返回值

```java
// 反例：吞异常
try {
    // ...
} catch (Exception e) {
    // 什么都不做 ❌ 出了问题完全不知道
}

// 正例：至少打印日志
try {
    // ...
} catch (Exception e) {
    e.printStackTrace();   // 或用日志框架记录
    throw e;               // 必要时往上抛
}
```
