# 大模型学习路线图（Java 开发者版）

> 适合有 Java 开发经验、零 AI 基础的学习者，6 个月系统掌握大模型核心技术。

---

## 第一个月：Python + 数学基础

### Python 语法（第 1 周）

你会 Java，Python 语法很快能上手，重点关注和 Java 的差异。

| 资源 | 说明 |
|------|------|
| [Python 官方教程](https://docs.python.org/zh-cn/3/tutorial/) | 中文版，挑着看，跳过你已经懂的 |
| [菜鸟教程 - Python3](https://www.runoob.com/python3/) | 速查语法用 |

### 核心库（第 2-3 周）

| 库 | 学什么 | 资源 |
|----|--------|------|
| NumPy | 矩阵运算、广播机制 | [NumPy 官方入门教程](https://numpy.org/doc/stable/user/quickstart.html) |
| PyTorch | 张量操作、自动求导、构建网络 | [PyTorch 官方 60 分钟教程](https://pytorch.org/tutorials/beginner/deep_learning_60min_blitz.html) |
| Pandas | 数据处理（后面做数据集会用到） | 用到再学，不用专门花时间 |

### 数学补基础（第 4 周，穿插进行）

不用系统学，理解这些概念就够：

| 主题 | 资源 | 重点 |
|------|------|------|
| 线性代数 | [3Blue1Brown 线性代数的本质](https://www.bilibili.com/video/BV1ys411472E/) | 矩阵乘法、特征值、向量空间的直觉 |

### 本月目标检验

- [ ] 能用 PyTorch 手写一个简单的神经网络（比如 MNIST 手写数字识别）
- [ ] 理解前向传播、反向传播、梯度下降的过程

## 第二个月：Transformer 原理

这是整个学习路线中最重要的一个月。

### 学习顺序

| 步骤 | 资源 | 说明 |
|------|------|------|
| 1. 先看课程建立直觉 | [李宏毅 2024 机器学习课程](https://www.bilibili.com/video/BV1TD4y137mP/) | 重点看 Self-Attention 和 Transformer 相关章节 |
| 2. 看最经典的讲解视频 | [Andrej Karpathy - Let's build GPT](https://www.youtube.com/watch?v=kCc8FmEb1nY) | **强烈推荐**，从零手写一个 GPT，2 小时讲透原理 |
| 3. 图解博客辅助理解 | [The Illustrated Transformer（中文翻译）](https://jalammar.github.io/illustrated-transformer/) | 可视化理解 Attention 机制 |
| 4. 读原论文 | [Attention is All You Need](https://arxiv.org/abs/1706.03762) | 读不懂没关系，带着前面的理解再看会清晰很多 |
| 5. 自己动手实现 | 跟着 Karpathy 的视频手写一遍 | 这一步最痛苦但收获最大 |

### 拓展阅读（有余力再看）

| 资源 | 说明 |
|------|------|
| [The Illustrated GPT-2](https://jalammar.github.io/illustrated-gpt2/) | 理解 GPT 系列的生成方式 |
| [nanoGPT 源码](https://github.com/karpathy/nanoGPT) | Karpathy 的极简 GPT 实现，代码很干净 |

### 本月目标检验

- [ ] 能向别人解释清楚 Self-Attention 的计算过程（Q、K、V 是什么）
- [ ] 能说清楚 Encoder 和 Decoder 的区别
- [ ] 自己从零实现过一个简化版 Transformer

## 第三个月：HuggingFace 生态 + 微调入门

### HuggingFace 入门（第 1-2 周）

| 资源 | 说明 |
|------|------|
| [HuggingFace 官方课程](https://huggingface.co/learn/nlp-course) | 免费，质量极高，必看 |
| [HuggingFace Transformers 文档](https://huggingface.co/docs/transformers) | 学会加载模型、Tokenizer、Pipeline |

重点掌握：
- 用 `pipeline` 快速调用模型
- 用 `AutoModel` / `AutoTokenizer` 加载模型
- 理解 Tokenizer 的工作原理（BPE 分词）

### 微调实战（第 3-4 周）

| 资源 | 说明 |
|------|------|
| [HuggingFace PEFT 文档](https://huggingface.co/docs/peft) | LoRA / QLoRA 微调指南 |
| [用 LoRA 微调 LLaMA 的教程](https://github.com/tloen/alpaca-lora) | 经典实战项目 |
| Google Colab 免费 T4 GPU | 前期够用，不用花钱 |

建议用 Qwen2 或 LLaMA 3 的小参数版本（7B/8B）做实验。

### 本月目标检验

- [ ] 能用 HuggingFace 加载开源模型做推理
- [ ] 成功用 LoRA 微调过一个模型（哪怕是很小的数据集）
- [ ] 理解全参微调 vs LoRA vs QLoRA 的区别和适用场景

## 第四个月：关键技术深入

### 核心论文和技术

按优先级排列，不用每篇都精读，理解在解决什么问题即可：

| 技术 | 要理解的核心问题 | 推荐资源 |
|------|-----------------|----------|
| 分词 (BPE/SentencePiece) | 文本怎么变成数字 | [Karpathy - Let's build the GPT Tokenizer](https://www.youtube.com/watch?v=zduSFxRajkE) |
| 位置编码 (RoPE) | 模型怎么知道词的顺序 | 搜索 "RoPE 旋转位置编码 详解" |
| KV Cache | 为什么推理时需要缓存 | 搜索 "KV Cache 推理加速 原理" |
| Flash Attention | 怎么让 Attention 更快更省显存 | [Flash Attention 论文](https://arxiv.org/abs/2205.14135) |
| RLHF / DPO | 模型怎么对齐人类偏好 | [InstructGPT 论文](https://arxiv.org/abs/2203.02151) + 李宏毅 RLHF 课程 |

### 了解主流模型架构

| 模型系列 | 要了解的 |
|----------|----------|
| GPT 系列 | Decoder-Only 架构，自回归生成 |
| LLaMA 系列 | Meta 开源，目前开源社区最活跃 |
| Qwen 系列 | 阿里开源，中文能力强 |
| DeepSeek 系列 | MoE 架构，性价比高 |

### 本月目标检验

- [ ] 能说清楚一个 LLM 从输入到输出的完整流程
- [ ] 理解 RLHF 为什么重要、大致怎么做的
- [ ] 知道主流开源模型的特点和差异

## 第五个月：RAG + 工程落地

### RAG（检索增强生成）（第 1-2 周）

| 资源 | 说明 |
|------|------|
| [LangChain 官方教程](https://python.langchain.com/docs/tutorials/) | 快速了解 RAG 流程 |
| [LlamaIndex 文档](https://docs.llamaindex.ai/) | 另一个主流 RAG 框架 |
| 向量数据库：Chroma / Milvus / FAISS | 选一个学会基本使用 |

RAG 核心流程：文档切分 → 向量化 → 存入向量数据库 → 检索 → 拼接 Prompt → 生成。

### 推理部署（第 3-4 周）

| 工具 | 说明 |
|------|------|
| [vLLM](https://github.com/vllm-project/vllm) | 高性能推理引擎，生产级 |
| [Ollama](https://ollama.com/) | 本地跑模型最简单的方式 |
| [llama.cpp](https://github.com/ggerganov/llama.cpp) | CPU 推理，了解即可 |

### 量化（穿插学习）

| 技术 | 说明 |
|------|------|
| GPTQ | 训练后量化，精度较好 |
| AWQ | 更新的量化方法 |
| GGUF | llama.cpp 使用的格式 |

理解"为什么要量化"（显存不够）和"量化会损失什么"就行。

### 本月目标检验

- [ ] 能搭建一个完整的 RAG 系统（基于自己的文档问答）
- [ ] 能用 vLLM 或 Ollama 部署一个开源模型
- [ ] 理解 4bit/8bit 量化的基本原理

## 最后的建议

1. **不要追求完美再开始下一步**——每个阶段 70% 掌握就往前走，后面会不断回来加深理解
2. **多写代码少看视频**——看 10 小时视频不如自己跑通 1 个项目
3. **记笔记**——用自己的话写技术博客或笔记，写不出来说明没真懂
4. **找同伴**——加一两个学习群，有人讨论进步会快很多
5. **接受不确定性**——这个领域变化极快，今天的最佳实践明天可能过时，保持学习能力比记住具体技术更重要
