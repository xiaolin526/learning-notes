## 怎么用这份手册答辩

答辩老师无非问三类问题：**"这是什么？""怎么做的？""为什么这么做？"**
所以本手册每个区块都按同一套"答辩四件套"来写，你照着背就行：

| 模块 | 作用 | 答辩时对应 |
|---|---|---|
| **① 效果** | 这块在页面上长什么样、能干什么 | 回答"这是什么" |
| **② 知识点** | 用到了哪些技术（CSS / JS / GSAP / Bootstrap） | 回答"用了什么技术" |
| **③ 代码逐行讲** | 关键代码一行一行说人话 | 回答"怎么做的" |
| **④ 一句话讲 + 追问** | 现场可以直接说出口的话术 + 老师可能追问与标准答案 | 回答"为什么"、应对深挖 |

**答辩小技巧**：不要逐行念代码，先说"效果"，再说"我用了 XX 技术实现"，老师感兴趣再展开"③"。把"④"里的追问提前练熟，能稳住场子。

## 第一章 · 先打地基（10 分钟看懂全局）

### 1.1 一个网页是怎么构成的？（盖房子比喻）

一个网页 = **三种文件**各司其职，可以用盖房子来理解：

| 技术 | 文件后缀 | 比喻 | 负责什么 |
|---|---|---|---|
| **HTML** | `.html` | 房子的**毛坯结构**（墙、门、窗的位置） | 网页上有哪些内容：标题、段落、图片、按钮 |
| **CSS** | `.css` | 房子的**装修**（刷漆、贴砖、挂灯） | 这些内容长什么样：颜色、大小、位置、动画 |
| **JavaScript** | `.js` | 房子的**电路和开关**（按一下灯就亮） | 交互行为：点击、滚动、鼠标悬停时发生什么 |

> **一句话答辩**："HTML 负责**结构**，CSS 负责**样式**，JavaScript 负责**交互**。三者分工明确，这也是前端开发最基本的'结构、表现、行为'三层分离思想。"

### 1.2 HTML 长什么样？（标签的概念）

HTML 由一对对**标签（tag）** 组成，像一个个盒子：

```html
<h1>敦煌</h1>          <!-- h1 是一级大标题 -->
<p>这是一段文字</p>     <!-- p 是段落 paragraph -->
<img src="图片.jpg">    <!-- img 是图片 image，src 指明图片地址 -->
<a href="千年丝路.html">点我跳转</a>  <!-- a 是链接，href 是目标地址 -->
<div>...</div>         <!-- div 是万能容器，用来分组、当"盒子" -->
```

- `<h1>` 是**开始标签**，`</h1>` 是**结束标签**，中间夹的是内容。
- 标签里还能写**属性**，比如 `class="site-nav"`，`class` 就像给这个盒子贴了个名字标签，方便 CSS 和 JS 找到它。

> **答辩高频词**：`class`（类名）。CSS 和 JS 都是靠 `class` 这个"名字"来精确控制某个元素的。

### 1.3 CSS 长什么样？（选择器 + 规则）

CSS 的写法是：**"选中谁 { 给它什么样式 }"**

```css
.site-nav {          /* .site-nav 表示"找到所有 class 叫 site-nav 的元素" */
  background: red;   /* 背景设为红色 */
  height: 64px;      /* 高度 64 像素 */
}
```

- 开头的 `.` 表示按 **class 名**找元素（这叫**选择器**）。
- 大括号里每行是"属性: 值;"，分号结尾。

### 1.4 JavaScript 长什么样？（让网页动起来）

JS 能在用户操作时改变页面。最常见的套路：

```js
// 找到那个按钮，给它装一个"被点击时要做的事"
button.addEventListener('click', function () {
  menu.classList.toggle('open');   // 点一下，给菜单加/去掉 open 这个 class
});
```

不懂没关系，记住一句话：**JS 的核心就是"找到元素 → 监听用户动作 → 改变元素"**。

### 1.5 本项目还借了三个"外部工具"

我们没全部自己手写，引入了三个成熟的库（别人写好的工具包）：

| 工具 | 是什么 | 本项目用它来做 |
|---|---|---|
| **Bootstrap** | 最流行的 CSS **框架**（一套现成的样式 + 组件） | 提供"栅格布局"和"轮播图"组件，省去自己造轮子 |
| **GSAP** | 专业的 JS **动画引擎**（业界标准） | 所有"随滚动播放"的高级动画 |
| **ScrollTrigger** | GSAP 的**滚动插件** | 让动画跟着鼠标滚动进度走（边滚边演） |

> **答辩一句话**："基础样式和交互我用原生 HTML/CSS/JS 实现；布局和轮播借助 **Bootstrap** 框架提高效率；复杂的滚动动画用业界标准的 **GSAP** 动画引擎及其 **ScrollTrigger** 插件完成。"

### 1.6 文件加载顺序为什么重要？

在 HTML 的 `<head>` 和 `<body>` 底部，文件是**从上到下依次加载**的，顺序不能乱。本项目的标准顺序是：

**CSS（在 head 里）**：
```text
Bootstrap 的 CSS  →  dunhuang.css（全局变量/通用样式）  →  本页专属 CSS
```
**为什么这个顺序**：后加载的 CSS 能**覆盖**先加载的。先铺 Bootstrap 的"地砖"，再用我们自己的 `dunhuang.css` 定义全站统一风格，最后用本页 CSS 做个性化微调——层层精修，互不打架。

**JS（在 body 底部）**：
```text
GSAP → ScrollTrigger → Bootstrap JS → 脚本.js（全局）→ 本页.js
```
**为什么放在底部**：JS 要操作页面元素，得等 HTML 结构先加载出来，所以放最后。**为什么这个顺序**：本页脚本依赖 GSAP，所以 GSAP 必须先到位。

> **老师可能追问**："如果把本页 CSS 写在 dunhuang.css 前面会怎样？"
> **答**："可能导致本页想覆盖全局的样式失效，因为 CSS 遵循'后来居上'的层叠规则，本页样式必须在全局样式之后加载才能正确覆盖。"

## 第二章 · 三页共用的"地基层"

> 这一章讲三个页面**都在用**的公共代码。吃透它，后面三页能省一半口舌。主要是 `dunhuang.css`（全局样式）和 `脚本.js`（全局交互）。

### 2.1 设计令牌：CSS 变量系统（`:root`）

#### ① 效果
整个网站的颜色、字体、圆角都**统一管理在一个地方**。想把"敦煌红"调浅一点？只改一行，全站三个页面同步生效。

#### ② 知识点
**CSS 自定义属性（CSS Variables / 设计令牌 Design Tokens）**。

#### ③ 代码逐行讲
在 `dunhuang.css` 最顶部：

```css
:root {
  --sand: #f4e8d1;     /* 戈壁沙：全站背景色 */
  --red: #8b2626;      /* 敦煌红：导航、大标题、按钮 */
  --gold: #d4af37;     /* 鎏金色：线条、边框、悬停发光 */
  --green: #2f6b5e;    /* 矿物绿：表头等小面积点缀 */
  --ink: #3a2e25;      /* 墨色：正文，比纯黑柔和 */
  --font-display: "Noto Serif SC", "KaiTi", serif;  /* 标题字体 */
  --radius-card: 18px; /* 卡片统一圆角 */
}
```

- `:root` 代表整个文档的"根"，在这里定义的变量**全站可用**。
- `--sand` 这种 `--` 开头的就是**变量**，相当于给颜色起了个外号。
- 用的时候写 `var(--sand)`：

```css
body { background: var(--sand); }   /* 背景 = 戈壁沙色 */
.btn { background: var(--red); }    /* 按钮 = 敦煌红 */
```

#### ④ 一句话讲 + 追问
> **一句话**："我把全站的主色调、字体、圆角都抽成 CSS 变量统一管理，这叫**设计令牌**。好处是风格统一、便于维护——改一处，三页同步更新。"

> **追问：CSS 变量和普通写死颜色（如直接写 `#8b2626`）有什么区别？**
> **答**："写死的话，全站几十处用到敦煌红，改色就要改几十遍还容易漏；用变量只改 `:root` 里一行。而且变量名 `--red` 比一串十六进制色值更可读，团队协作更清晰。"

> **追问：为什么背景用沙色不用纯白？**
> **答**："这是设计规范里的刻意选择。敦煌主题追求古朴温润，纯白太刺眼且没有历史感，戈壁沙色更贴合大漠氛围，长时间阅读也更护眼。"

### 2.2 全站导航 `.site-nav`（吸顶 + 发光 + 汉堡菜单）

#### ① 效果
一条**敦煌红横条永远吸在屏幕顶部**，滚动页面它不动；鼠标移到菜单项上发金光；当前所在页的菜单常亮；手机上收成"☰"汉堡菜单点开。

#### ② 知识点
固定定位 `position: fixed`、弹性盒 `flex`、CSS 变量、过渡 `transition`、`:hover` 悬停、`box-shadow` 发光、媒体查询 `@media` 响应式、JS 切换 class、无障碍 `aria-*`。

#### ③ 代码逐行讲
```css
.site-nav {
  position: fixed;       /* 固定定位：钉在屏幕上，不随页面滚动 */
  top: 0; left: 0; right: 0;  /* 顶到最上方，左右拉满 */
  z-index: 100;          /* 图层高度 100，压在所有内容之上 */
  display: flex;                      /* 弹性盒布局 */
  justify-content: space-between;     /* 两端对齐：品牌挤左，菜单挤右 */
  align-items: center;                /* 垂直居中 */
  height: 64px;
  background: linear-gradient(to bottom, var(--red), var(--red-deep)); /* 上下渐变红 */
  border-bottom: 2px solid var(--gold);  /* 底部一条金线 */
}
```

**关键配套**——因为导航 `fixed` 后会"飘"在页面上方盖住正文，所以正文区要让出空间：
```css
.page { padding-top: 64px; }   /* 正文顶部留出一个导航的高度 */
```

**鼠标悬停发金光**：
```css
.site-nav ul a {
  border-radius: 999px;          /* 超大圆角 = 药丸形状 */
  transition: all 0.35s ease;    /* 所有变化用 0.35 秒平滑过渡 */
}
.site-nav ul a:hover,
.site-nav ul a.active {          /* 鼠标悬停 或 当前页 */
  color: var(--gold);
  box-shadow: 0 0 12px var(--gold-soft) inset,  /* 内发光 */
              0 0 10px var(--gold-soft);         /* 外发光 */
}
```

**手机端变汉堡菜单**（媒体查询）：
```css
@media (max-width: 860px) {     /* 当屏幕宽度 ≤ 860 像素时 */
  .site-nav ul { display: none; }    /* 默认把横排菜单藏起来 */
  .site-nav ul.open { display: flex; }  /* 加了 open 类才显示 */
  .nav-toggle { display: block; }    /* 露出"☰"汉堡按钮 */
}
```

**点击汉堡的 JS**（在 `脚本.js`）：
```js
toggle.addEventListener('click', function () {
  var open = menu.classList.toggle('open');  // 切换 open 类：有就去掉，没有就加上
  toggle.setAttribute('aria-expanded', open ? 'true' : 'false'); // 同步无障碍状态
});
```

#### ④ 一句话讲 + 追问
> **一句话**："导航用 `position: fixed` 实现吸顶，弹性盒布局让品牌和菜单两端对齐，悬停用 `box-shadow` 做鎏金发光。移动端通过媒体查询折叠成汉堡菜单，由 JS 切换 `open` 类来展开收起。"

> **追问（高频！）：为什么用了 `fixed` 还要给 `.page` 加 `padding-top`？**
> **答**："`fixed` 会让导航**脱离文档流**，不再占据高度，正文会顶到屏幕最上方被导航盖住。加一个等于导航高度（64px）的 `padding-top`，正好把正文内容往下推，避免被遮挡。"

> **追问：`aria-expanded`、`aria-label` 是干嘛的？**
> **答**："这是**无障碍（Accessibility）** 属性，给视障用户的读屏软件用。`aria-label` 告诉它这个按钮叫'展开导航菜单'，`aria-expanded` 告诉它菜单当前是展开还是收起。体现了对所有用户的关怀。"

> **追问：`z-index: 100` 是什么？**
> **答**："控制元素的**堆叠层级**，数字越大越靠上层。导航设 100 保证它盖在所有正文内容之上，不会被图片或卡片挡住。"

### 2.3 页脚 `.site-footer`（画卷落款）

#### ① 效果
页面最底部的**深色页脚**：左边一枚"大漠遗珠"朱砂印章 + 品牌名，右边两列链接，底部版权信息。三个页面共用。

#### ② 知识点
弹性盒布局、`flex-wrap` 换行、CSS 变量、媒体查询。

#### ③ 代码逐行讲
```css
.site-footer {
  background-color: #0d0b0a;   /* 极深暖黑，沉淀收束 */
  padding: 8vh 6vw 4vh;        /* vh/vw 是相对屏幕高/宽的单位，自适应 */
}
.footer-container {
  display: flex;
  justify-content: space-between;  /* 左右两块拉开 */
  flex-wrap: wrap;                 /* 空间不够时自动换行 */
}
.seal {                       /* 朱砂印章 */
  width: 56px; height: 56px;
  border: 2px solid #8b2522;  /* 朱砂红边框 */
  display: flex;
  align-items: center; justify-content: center;  /* 文字居中 */
}
```
手机端一列堆叠：
```css
@media (max-width: 768px) {
  .footer-container { flex-direction: column; }  /* 改成竖向排列 */
}
```

> **代码里有个工程亮点值得讲**：原本这套页脚样式只写在 `首页.css` 里，导致其它页面必须额外引入整份首页样式才能显示页脚。后来**收口到 `dunhuang.css` 全局**，任何页面只要引入 `dunhuang.css` 就能用。

#### ④ 一句话讲 + 追问
> **一句话**："页脚用弹性盒做两端对齐布局，`flex-wrap` 保证窄屏自动换行，移动端切换成纵向堆叠。我特意把它从单页样式**收口到全局**，避免代码重复和样式依赖。"

> **追问：`vh`、`vw` 是什么单位？**
> **答**："`vh` 是视口高度的百分之一，`vw` 是视口宽度的百分之一。`8vh` 就是屏幕高度的 8%。好处是**自适应**——不同尺寸屏幕上间距比例一致。"

### 2.4 全局脚本 `脚本.js`（两件事）

`脚本.js` 三个页面都引入，只干两件全局的事：

**事一：滚动浮现（元素滚到视野里才淡入）**
```js
var io = new IntersectionObserver(function (entries) {
  entries.forEach(function (entry) {
    if (entry.isIntersecting) {        // 如果元素进入了视野
      entry.target.classList.add('in'); // 给它加 'in' 类（触发 CSS 淡入动画）
      io.unobserve(entry.target);       // 加完就不再观察它（只播一次）
    }
  });
}, { threshold: threshold });

// 观察所有带 reveal 类的元素
document.querySelectorAll('.reveal, .reveal-left, .reveal-right')
  .forEach(function (el) { io.observe(el); });
```

- `IntersectionObserver`（**交叉观察器**）是浏览器自带的工具，能高效地"监视某个元素有没有进入屏幕可视区域"。
- 配套 CSS：元素默认透明下移（`.reveal`），加上 `.in` 后回正（淡入上移）。

**事二：移动端汉堡菜单开关**（见 2.2 已讲）。

#### ④ 一句话讲 + 追问
> **一句话**："全局脚本用 **IntersectionObserver** 实现滚动浮现——元素滚进视野才淡入，且每个只触发一次。相比传统监听 scroll 事件，它由浏览器底层优化，**性能更好**。"

> **追问：为什么不用监听 scroll 事件来判断元素位置？**
> **答**："`scroll` 事件触发极其频繁（滚一下触发几十上百次），每次都要手动计算元素位置，性能差。`IntersectionObserver` 是浏览器原生的异步观察器，只在元素进出视野的瞬间回调一次，**更高效也更省电**。"

> **追问：`threshold` 是什么？**
> **答**："触发阈值，表示元素**露出多少比例**才算'进入视野'。比如 `0.18` 就是露出 18% 才触发淡入。各页面通过 HTML 上的 `data-reveal-threshold` 自定义灵敏度。"

### 2.5 通用动画约定 `.reveal` 与"减弱动效"无障碍

`dunhuang.css` 里定义了一套**出场动效**约定，三页通用：
```css
.reveal {
  opacity: 0;                    /* 初始透明 */
  transform: translateY(28px);   /* 初始下移 28px */
  transition: opacity 1.4s ease, transform 1.4s ease;  /* 平滑过渡 */
}
.reveal.in {       /* JS 加上 in 后 */
  opacity: 1;
  transform: none; /* 回到原位，呈现"淡入上浮"效果 */
}
```

**无障碍兜底**（重要加分项）：
```css
@media (prefers-reduced-motion: reduce) {  /* 用户在系统里开了"减少动态效果" */
  .reveal, .reveal-left, .reveal-right {
    opacity: 1 !important;
    transform: none !important;
    transition: none !important;    /* 直接显示，不做任何动画 */
  }
}
```

> **答辩加分**："我用 `prefers-reduced-motion` 媒体查询做了无障碍兜底——如果用户因前庭功能障碍等在系统里开启了'减少动态效果'，所有动画会自动关闭，内容直接呈现。这是对**晕动症用户**的关怀，也是现代前端的最佳实践。"

### 2.6 命名空间：怎么防止样式打架？

三个页面各有一套专属 CSS，怎么保证它们和 Bootstrap、和彼此不冲突？答案是**给类名加前缀**：

| 页面 | 前缀 | 例子 |
|---|---|---|
| 千年丝路 | `sl-`（silk road） | `.sl-hero` `.sl-epoch` `.sl-fig` |
| 光影敦煌 | `gy-`（光影） | `.gy-card` `.gy-frame` `.gy-img` |

> **答辩一句话**："我给每个页面的类名加了专属前缀，比如千年丝路用 `sl-`、光影敦煌用 `gy-`，相当于给样式划分了**命名空间**，避免和 Bootstrap 的类名或其它页面的样式互相覆盖、打架。"

## 第三章 · 首页 首页.html 逐块详解

> 首页是动画最密集的一页，共四大区块。它用了**两套动画体系**：纯 CSS 动画（Hero 文字入场、掌灯夜探）和 **GSAP + ScrollTrigger** 滚动动画（时光卷轴、横向长卷）。

### 3.1 全屏电影感 Hero · 壁画多层视差

#### ① 效果
打开页面是**整屏壁画大图**。往下滚时，背景图**纹丝不动**、前景的飞天/祥云/石头**跟着页面走**，制造出"景深"的视差错觉。标题"敦煌"等文字**错峰依次升起淡入**；底部祥云与飞天**轻轻飘动**。

#### ② 知识点
- **视差滚动**：`background-attachment: fixed`
- **绝对定位**叠放多层前景图（`m-layer`）
- **CSS 关键帧动画** `@keyframes` + `animation-delay` 错峰
- `clamp()` 响应式字号
- 无障碍：`prefers-reduced-motion`

#### ③ 代码逐行讲

**视差的核心就一行**（在 `.hero`）：
```css
.hero {
  height: 100vh;                       /* 占满整屏高度 */
  background-image: url("../images/背景.jpg");
  background-size: cover;              /* 图片铺满不留白 */
  background-attachment: fixed;        /* ★关键：背景相对"屏幕"固定，不随页面滚 */
}
```
> `background-attachment: fixed` 的意思是：背景图钉在**浏览器窗口**上，而不是钉在元素上。于是你滚动页面时，前景内容往上走、背景留在原地——这就是最简单的视差。

**iOS 兼容兜底**（细节加分）：
```css
@media (hover: none), (max-width: 768px) {
  .hero { background-attachment: scroll; }  /* 触屏设备降级，因 fixed 在 iOS Safari 会卡顿失效 */
}
```

**多层前景**（HTML 里每个 `m-layer` 是一层，用 `style` 内联定位）：
```html
<div class="m-layer" style="left:-3%; top:4%; width:32%; z-index:2">
  <div class="m-amb"><img src="../images/祥云1.png" alt="祥云·左"></div>
</div>
```
```css
.m-layer { position: absolute; }  /* 绝对定位：用 left/top/right/bottom 精确摆放 */
```
- `z-index` 数字越大越靠前（祥云 z-index:2，飞天人物 z-index:4 在更前面）。

**文字错峰入场**：
```css
.rise {
  opacity: 0;
  transform: translateY(34px);          /* 初始：透明 + 下移 34px */
  animation: rise 2.2s ease forwards;   /* 播放 rise 动画，forwards=停在最后一帧 */
}
.rise.d1 { animation-delay: 0.4s; }   /* 第1个延迟 0.4 秒开始 */
.rise.d2 { animation-delay: 1s; }     /* 第2个延迟 1 秒，形成"错峰" */
.rise.d3 { animation-delay: 1.6s; }
.rise.d4 { animation-delay: 2.2s; }
@keyframes rise {       /* 定义动画：从初始状态变到下面这个终态 */
  to { opacity: 1; transform: none; }   /* 终态：完全显现，回到原位 */
}
```
- `@keyframes rise` 定义了一段动画"从透明下移 → 不透明回正"。
- 四个文字用 `d1~d4` 设置不同 `animation-delay`，于是它们**一个接一个升起**。

**响应式字号 `clamp()`**：
```css
.hero-text h1 { font-size: clamp(44px, 8vw, 96px); }
```
> `clamp(最小, 理想, 最大)`：字号理想值是屏幕宽度的 8%，但不会小于 44px、不会大于 96px。**一行代码搞定大小屏自适应**，不用写一堆媒体查询。

#### ④ 一句话讲 + 追问
> **一句话**："Hero 区用 `background-attachment: fixed` 实现背景固定、前景滚动的**视差效果**；前景祥云飞天用绝对定位分层叠放，靠 `z-index` 控制前后；标题文字用 `@keyframes` 动画配合不同的 `animation-delay` 做**错峰升起**入场。字号用 `clamp()` 自适应。"

> **追问（高频）：视差效果是怎么实现的？**
> **答**："核心是给背景设 `background-attachment: fixed`，让背景图相对浏览器窗口固定。滚动时前景元素往上移动，而背景停在原处，两者产生相对位移，人眼就感知到了**景深**。我还为 iOS 触屏做了降级处理，因为该属性在移动 Safari 上有兼容问题。"

> **追问：`@keyframes` 和 `transition` 有什么区别？**
> **答**："`transition` 只能定义'起点→终点'两个状态之间的过渡，需要某个触发（如 hover）。`@keyframes` 能定义**多个关键帧**的复杂动画，且可以自动播放、循环、延迟。Hero 文字入场用 keyframes，导航 hover 用 transition。"

> **追问：`forwards` 是什么意思？**
> **答**："`animation-fill-mode: forwards` 让动画**停留在最后一帧**。不写的话，文字播完入场动画会'弹回'初始的透明状态，写了它才会保持显现。"

### 3.2 时光卷轴 · GSAP 时间轴（最复杂、最值得细讲）

#### ① 效果
一个**羊皮纸质感**的区域，中间一条**金色竖线随滚动从上往下生长**；左右交叉排列的图文节点（左文右图、左图右文）随滚动**淡入**；文字里的数字（366、735、45000）**从 0 滚动跳到目标值**；结尾的朱砂印章**盖章式**弹入。

#### ② 知识点
- **GSAP**（动画引擎）+ **ScrollTrigger**（滚动插件）
- CSS **Grid 网格布局**（三列：文 / 圆点 / 图）
- `mix-blend-mode: multiply` 正片叠底（图片融入纸张）
- CSS `mask-image` 边缘羽化
- JS 数字递增动画
- 减弱动效兜底

#### ③ 代码逐行讲

**布局：用 Grid 做三列对称**
```css
.scroll-section .node {
  display: grid;
  grid-template-columns: 1fr 44px 1fr;  /* 三列：左半 | 中间44px放圆点 | 右半 */
  align-items: center;
}
.node-left  .node-content { grid-column: 1; text-align: right; }  /* 左节点：文字在第1列 */
.node-left  .node-image   { grid-column: 3; }                     /*         图片在第3列 */
.node-right .node-content { grid-column: 3; }  /* 右节点反过来，形成左右交叉 */
.node-right .node-image   { grid-column: 1; }
```
> `1fr` 是"一份"的意思，`1fr 44px 1fr` 表示左右两列平分剩余空间、中间固定 44px。`grid-column: 1` 表示"放在第几列"。通过给左右节点分配相反的列，实现 **Z 字形交叉排版**。

**金线生长（GSAP scrub）**——在 `首页.js`：
```js
gsap.registerPlugin(ScrollTrigger);   // 注册滚动插件

gsap.to('.timeline-line', {           // 让金线动起来
  scaleY: 1,                          // 纵向从 0 拉伸到 1（CSS 里初始 scaleY(0)）
  ease: 'none',
  scrollTrigger: {
    trigger: '.timeline',             // 以时间轴区域为触发参照
    start: 'top 72%',                 // 区域顶部到达视口 72% 处时开始
    end: 'bottom 78%',                // 区域底部到达 78% 处时结束
    scrub: true                       // ★关键：动画进度"绑定"滚动条，边滚边长
  }
});
```
配套 CSS 让金线初始为 0 高：
```css
.timeline-line { transform: translateX(-50%) scaleY(0); transform-origin: top center; }
```
> **`scrub: true` 是这段的灵魂**：它把动画进度和滚动进度绑死——你往下滚多少，金线就长多少；往回滚，它就缩回去。

**数字从 0 跳到目标值**：
```js
function countUp(el){
  const target = parseInt(el.dataset.target, 10);  // 读 HTML 里 data-target="366"
  const obj = { v: 0 };                             // 造个临时对象，值从 0 开始
  gsap.to(obj, {
    v: target, duration: 1.6,                       // 1.6 秒内把 v 从 0 变到 target
    onUpdate(){ el.textContent = Math.floor(obj.v).toLocaleString('en-US'); }
    // 每一帧都把当前值写回页面，toLocaleString 自动加千分位逗号（45,000）
  });
}
```
HTML 里：
```html
公元 <span class="count" data-target="366">0</span> 年
```
> `data-target` 是**自定义数据属性**（`data-` 开头），把"目标数字"存在 HTML 上，JS 用 `el.dataset.target` 读出来。这是 HTML 和 JS 之间传值的标准做法。

**图片融入纸张**：
```css
.node-image {
  mix-blend-mode: multiply;          /* 正片叠底：图片与背景纸色混合，去掉白底显旧 */
  filter: sepia(0.35) contrast(1.1) brightness(0.95);  /* 泛黄做旧滤镜 */
  -webkit-mask-image: radial-gradient(ellipse 74% 80% at center,
                       rgba(0,0,0,1) 55%, rgba(0,0,0,0) 100%);  /* 边缘羽化淡出 */
}
```
> `mix-blend-mode: multiply`（正片叠底）让图片的白色部分变透明、与纸张融为一体；`mask-image` 用一个中心实、边缘虚的渐变当"蒙版"，让图片四周**羽化淡出**，没有生硬的方框边界。

**朱砂印章盖章入场**：
```js
gsap.from(seal, {
  scale: 1.6, opacity: 0, rotation: -18,  // 从"放大1.6倍+透明+逆时针18度"
  duration: 0.5, ease: 'power4.out',      // 0.5秒快速收束
  scrollTrigger: { trigger: '.coda-wrapper', start: 'top 80%', once: true }  // 只盖一次
});
```
> `gsap.from` 表示"**从**这个状态变到**当前**状态"。所以印章从"大、透明、歪着"快速变成"正常大小、清晰、摆正"，模拟**啪地盖下印章**的感觉。`once: true` 保证只播一次。

#### ④ 一句话讲 + 追问
> **一句话**："时光卷轴用 **CSS Grid** 做左右交叉的图文排版，用 **GSAP + ScrollTrigger** 驱动滚动动画：金色时间轴线用 `scrub` 模式**随滚动进度生长**，数字用 GSAP 从 0 递增到目标值，图片用 `mix-blend-mode` 正片叠底融入纸张，印章用 `gsap.from` 做盖章入场。"

> **追问（必考）：`scrub: true` 是什么效果？和普通动画区别？**
> **答**："普通动画一旦触发就**自动播完**。`scrub: true` 是把动画进度**绑定到滚动条**——滚动到哪，动画就播到哪，往回滚动画也倒退。所以金线是'边滚边长'，给人**亲手展开画卷**的掌控感。"

> **追问：`gsap.to` 和 `gsap.from` 区别？**
> **答**："`gsap.to` 是'从当前状态**到**指定状态'，`gsap.from` 是'**从**指定状态到当前状态'。入场动画常用 `from`，因为我们只需描述'它从哪个奇怪状态飞进来'，终点就是它在页面上的正常样子。"

> **追问：Grid 和 Flex 都能布局，为什么这里用 Grid？**
> **答**："Flex 是**一维**布局（一行或一列）。这里需要'左半、中线、右半'三列严格对齐，且左右节点要在固定列之间切换位置，这是**二维网格**问题，用 Grid 的 `grid-template-columns` 和 `grid-column` 控制最直接。"

> **追问：`data-target` 这种 `data-` 属性是什么？**
> **答**："HTML5 的**自定义数据属性**，用来在 HTML 元素上存放额外数据。JS 通过 `元素.dataset.target` 读取。这里我把每个数字的目标值存在 `data-target` 上，JS 读出来做递增动画，实现了**数据和逻辑的分离**。"

### 3.3 掌灯夜探石窟 · 纯 CSS 互动（不用一行 JS！）

#### ① 效果
三扇并排的**漆黑洞窟门**，默认压暗模糊。鼠标移到哪一扇，**哪扇就被'点亮'**——变清晰、放大占更多宽度、浮起金光和文字介绍，旁边两扇则收窄变暗。全程**不用 JavaScript**。

#### ② 知识点
- 弹性盒 `flex` 的**伸缩比例**（`flex` 数值）
- `:hover` / `:focus-within` 悬停聚焦
- 多层 `::before`、蒙版、`radial-gradient` 探照灯光晕
- `transition` 平滑过渡
- `backdrop-filter` 背景模糊

#### ③ 代码逐行讲

**三扇门平分宽度**：
```css
.grotto-explorer { display: flex; height: 78vh; }
.grotto-gate {
  flex: 1;                  /* 三扇默认各占 1 份，平分 */
  transition: flex 0.5s;    /* flex 变化时用 0.5 秒平滑过渡 */
}
```

**鼠标悬停时重新分配宽度**（纯 CSS 的精髓）：
```css
.grotto-explorer:hover .grotto-gate { flex: 0.6; }      /* 鼠标进入容器，所有门先缩到 0.6 */
.grotto-explorer .grotto-gate:hover { flex: 2.4; }      /* 但被指到的那扇放大到 2.4 */
```
> 这两行是关键魔法：鼠标进入整个容器时，**所有门**先变窄（0.6）；而**正被悬停的那扇**单独放大（2.4）。因为有 `transition: flex 0.5s`，宽度变化是平滑滑动的，于是产生"聚焦展开"的效果——**全程没用 JS**。

**点亮：去掉黑色蒙版 + 浮现金光**：
```css
.grotto-gate .gate-mask { background: #000; opacity: 0.85; }  /* 默认盖一层85%黑幕 */
.grotto-gate:hover .gate-mask { opacity: 0; }                 /* 悬停时黑幕消失 → 点亮 */

.grotto-gate::before {                       /* 用伪元素放一张模糊暗背景 */
  filter: brightness(0.5) blur(3px);         /* 默认压暗+模糊 */
}
.grotto-gate:hover::before {
  filter: brightness(1) blur(0);             /* 悬停时恢复清晰明亮 */
}
```

**文字介绍滑入**：
```css
.gate-info { opacity: 0; transform: translateY(30px); }  /* 默认透明+下移藏起 */
.grotto-gate:hover .gate-info { opacity: 1; transform: translateY(0); }  /* 悬停滑入 */
```

#### ④ 一句话讲 + 追问
> **一句话**："掌灯夜探是一个**纯 CSS** 实现的交互画廊，没用任何 JS。核心是利用弹性盒的 `flex` 伸缩比例——鼠标悬停容器时所有门收窄，被悬停的那扇单独放大，配合 `transition` 形成平滑的'聚焦展开'。点亮效果靠悬停时移除黑色蒙版、恢复图片清晰度来实现。"

> **追问（亮点！）：这个交互为什么不用 JavaScript？纯 CSS 有什么好处？**
> **答**："因为它的所有状态切换都能由 `:hover` 伪类触发，无需监听事件。纯 CSS 实现的好处是：**性能好**（浏览器原生处理，不走 JS 线程）、**代码简洁**、即使 JS 加载失败也能正常工作（健壮性强）。这体现了'能用 CSS 就不用 JS'的优化原则。"

> **追问：`:focus-within` 是什么？为什么要和 `:hover` 一起写？**
> **答**："`:hover` 只对鼠标有效，`:focus-within` 表示'内部有元素获得焦点'，用**键盘 Tab 切换**时也能触发。两个一起写，保证鼠标用户和键盘用户都能用，是无障碍考虑。"

> **追问：`::before` 伪元素是什么？**
> **答**："伪元素能在不增加 HTML 标签的情况下，'凭空'生成一个装饰用的子元素。这里用 `::before` 放模糊暗背景层，用 `::after`/蒙版做光晕，让 HTML 保持干净。"

### 3.4 千年画卷 · GSAP 横向长卷（pin 钉住 + 横向平移）

#### ① 效果
滚到这一区，页面会**被"钉住"不再纵向滚**，转而把一排壁画**横向地从右往左推过去**，像展开一幅长卷。推到头后，页面才继续正常往下滚。

#### ② 知识点
- GSAP **ScrollTrigger 的 `pin`（钉住）+ `scrub`（绑定滚动）**
- "纵向滚动驱动横向位移"的经典技法
- `width: max-content` 让容器等于内容总宽
- 动态计算位移距离 + 窗口变化时重算

#### ③ 代码逐行讲

**CSS：把面板排成一长横排**
```css
.mural-scroll-section { height: 100vh; overflow: hidden; }  /* 占满一屏，藏住溢出 */
.mural-wrapper {
  display: flex;          /* 所有面板横向排列 */
  width: max-content;     /* ★宽度=所有子面板宽度之和（可以远超屏幕宽） */
}
.mural-panel.img-panel img {
  height: 100%; width: auto;  /* 图片高度固定、宽度自适应，防止变形 */
}
```

**JS：把"纵向滚动"翻译成"横向移动"**——在 `首页.js`：
```js
function getScrollAmount() {
  return -(muralWrapper.scrollWidth - window.innerWidth);  // 需要左移的总距离=内容总宽-屏幕宽
}
const tween = gsap.to(muralWrapper, {
  x: getScrollAmount,    // 把整排面板沿 X 轴左移这么多
  ease: "none"           // 匀速，不加缓动（否则滚起来发飘）
});
ScrollTrigger.create({
  trigger: muralSection,
  animation: tween,
  pin: true,             // ★钉住：到这区域时把它固定在屏幕上
  scrub: 1,              // ★把横移进度绑定滚动条（1=有1秒平滑跟随）
  start: "top top",      // 区域顶部贴到视口顶部时开始
  end: () => `+=${muralWrapper.scrollWidth - window.innerWidth}`,  // 纵向预留=横向行程
  invalidateOnRefresh: true   // 窗口尺寸变化时重新计算
});
window.addEventListener('load', () => ScrollTrigger.refresh());  // 图片加载完后重算尺寸
```

> **机制拆解**（这段答辩最容易被深挖，务必理解）：
> 1. `pin: true` 把这一屏**钉在原地**，你继续滚鼠标，页面不往下走。
> 2. 你滚动的"力"被 `scrub` 转交给横向平移动画，于是壁画**横着走**。
> 3. `end` 设为"内容总宽 − 屏幕宽"，意思是**横向走多远，就预留多少纵向滚动距离**。走完了，pin 解除，页面继续正常下滚。
> 4. 图片是"高度100%、宽度自适应"，`DOMContentLoaded` 时还没加载完，量到的总宽偏小会算错距离——所以 `load` 后 `ScrollTrigger.refresh()` 强制**重算一次**。

#### ④ 一句话讲 + 追问
> **一句话**："横向长卷用 GSAP ScrollTrigger 的 `pin` 把区域钉在屏幕上，再用 `scrub` 把用户的**纵向滚动**转换成壁画的**横向平移**，实现'滚鼠标=展长卷'。横向行程通过 `内容总宽-视口宽` 动态计算，并在图片加载完成后刷新重算，避免出现空白条。"

> **追问（必考）：纵向滚动是怎么变成横向移动的？**
> **答**："关键是 `pin: true` 把区域固定，使页面无法纵向滚动；同时 ScrollTrigger 的 `scrub` 把滚动进度绑定到一个'横向平移'的动画上。所以用户以为在纵向滚，实际驱动的是壁画横向走。横向走完，pin 自动解除，恢复正常滚动。"

> **追问：`width: max-content` 起什么作用？**
> **答**："让 `.mural-wrapper` 的宽度等于**所有子面板宽度之和**，而不是被屏幕宽度限制。这样整排壁画才能'排成一条远超屏幕的长队'，给横向平移留出行程。"

> **追问：为什么要在 `load` 时再 `refresh()` 一次？**
> **答**："因为壁画图片设的是'高度100%、宽度自适应'，在 DOM 刚构建好时图片还没下载完，此时测到的总宽度偏小，会导致预留的滚动距离算错、出现白条。等图片真正加载完后调用 `ScrollTrigger.refresh()` 重新测量，尺寸就归位了。"

### 4.2 鎏金长河 · 滚动驱动驿道时间轴（本页核心）

#### ① 效果
左侧一条**虚线古道**，随滚动有一条**实线鎏金路从上往下生长**；每个朝代是一个"驿站"——左边竖排年号 + 古道节点，右边一张史事卡片。卡片随滚动**从右滑入**，鼠标悬停时卡片**抬起、图片由做旧黑白变焕彩**，对应的节点和年号也**同步点亮**。

#### ② 知识点
- GSAP `scrub` 实线生长 + 卡片 `scrollTrigger` 滑入
- **绝对定位**驿道与节点
- `repeating-linear-gradient` 画虚线
- `writing-mode: vertical-rl` **竖排文字**
- 弹性盒（轨 | 卡 并排，图 | 文 并排）
- CSS `filter` 做旧/焕彩 + **兄弟选择器联动**

#### ③ 代码逐行讲

**虚线古道 + 生长实线**（两条叠在一起）：
```css
.sl-route { position: absolute; top: 0; bottom: 0; left: 112px; width: 2px; }
.sl-route__base {     /* 静态虚线 */
  background: repeating-linear-gradient(to bottom,
    var(--gold-soft) 0 6px, transparent 6px 15px);  /* 金6px+透明9px循环=虚线 */
}
.sl-route__progress { /* 生长实线 */
  background: linear-gradient(to bottom, var(--gold), var(--red));
  transform: scaleY(0);       /* 初始高度为 0 */
  transform-origin: top;      /* 从顶部开始长 */
}
```
```js
gsap.to(".sl-route__progress", {
  scaleY: 1, ease: "none",
  scrollTrigger: { trigger: ".sl-river", start: "top 70%", end: "bottom 80%", scrub: true }
});  // 随滚动把实线从 scaleY(0) 长到 scaleY(1)
```
> `repeating-linear-gradient`（重复线性渐变）是画**虚线**的技巧：金色 6px、透明 9px，不断重复就成了虚线。生长实线则盖在虚线上，用 `scrub` 随滚动长出来，营造"路在脚下延伸"。

**竖排年号**：
```css
.sl-epoch__year {
  writing-mode: vertical-rl;     /* 文字竖排，从右到左 */
  text-orientation: upright;     /* 每个字保持正立（不躺倒） */
}
```
> `writing-mode: vertical-rl` 让"前138""盛唐"这些年号**竖着排**，呼应古籍竖排版式，是本页独有的中式排版细节。

**卡片滑入 + 节点弹出**：
```js
gsap.utils.toArray(".sl-epoch").forEach(function (epoch) {
  gsap.from(epoch.querySelector(".sl-epoch__card"), {
    x: 60, opacity: 0, duration: 0.9,    // 从右侧60px外透明滑入
    scrollTrigger: { trigger: epoch, start: "top 80%", toggleActions: "play none none reverse" }
  });
});
```
> `gsap.utils.toArray(...).forEach` 是**遍历所有驿站**，给每个单独绑一个滚动触发，所以它们各自滑入。`toggleActions: "play none none reverse"` 表示"进入时播放、离开时倒放"，来回滚都有动效。

**悬停焕彩 + 兄弟联动**（CSS 细节亮点）：
```css
.sl-epoch__img { filter: sepia(.4) saturate(.7) brightness(.82); }  /* 默认做旧泛黄 */
.sl-epoch__card:hover .sl-epoch__img { filter: none; transform: scale(1.05); }  /* 悬停焕彩+推近 */
.sl-epoch:hover .sl-epoch__node { transform: translateX(-50%) scale(1.25); }    /* 节点同步放大 */
.sl-epoch:hover .sl-epoch__year { color: var(--gold); }                          /* 年号同步变金 */
```
> 鼠标悬停卡片时，不仅卡片本身焕彩，连**旁边的古道节点和年号也一起点亮**——这是通过"悬停父级 `.sl-epoch` → 影响其内部子元素"实现的**联动反馈**，细节很考究。

#### ④ 一句话讲 + 追问
> **一句话**："鎏金长河是本页核心。虚线古道用 `repeating-linear-gradient` 绘制，鎏金实线用 GSAP `scrub` 随滚动生长；每个朝代用弹性盒做'年号轨 + 史事卡'并排，卡片用 ScrollTrigger 从右滑入。竖排年号用 `writing-mode` 实现。悬停时图片由做旧 `filter` 焕彩，并联动点亮节点与年号。"

> **追问：虚线是怎么用 CSS 画的？**
> **答**："用 `repeating-linear-gradient` 重复线性渐变——指定'金色 6px、透明 9px'然后无限重复，就形成了竖直虚线。这样不用切图，纯 CSS 实现，还能用变量统一颜色。"

> **追问：图片'黑白变彩色'的焕彩效果怎么做的？**
> **答**："用 CSS `filter` 滤镜。默认给图片加 `sepia`（泛黄）+ 降 `saturate`（去饱和）+ 降 `brightness`（压暗）做成做旧效果；悬停时把 `filter` 设为 `none` 恢复原图，配 `transform: scale` 轻微推近，配合 `transition` 平滑过渡，就有了'沉睡的历史被唤醒'的感觉。"

> **追问：悬停一张卡片，旁边的年号和节点也亮了，怎么做的？**
> **答**："利用 CSS 的**后代选择器联动**：当鼠标悬停在父容器 `.sl-epoch` 上时，用 `.sl-epoch:hover .sl-epoch__node` 这样的选择器同时改变其内部子元素的样式，实现一处悬停、多处响应的联动反馈。"

### 4.3 丝路之数 · 不等分弹性盒 + 数字计数

#### ① 效果
一条**深色大漆色带**，里面四个数字（7000+ 公里、2100 年、735 窟、50000+ 卷）。第一个"里程"格子**比其它三个更宽**。数字滚进视野时**从 0 跳到目标值**。

#### ② 知识点
- 弹性盒的**不等分**（`flex` 三个值）
- `flex-wrap` 换行响应式
- GSAP 计数 + ScrollTrigger 触发
- 伪元素 `::before` 画分隔线

#### ③ 代码逐行讲

**不等分弹性盒**：
```css
.sl-figures__inner { display: flex; flex-wrap: wrap; }
.sl-fig { flex: 1 1 230px; }        /* 普通格子：可放大可缩小，基准宽 230px */
.sl-fig--lead { flex: 1.5 1 300px; } /* ★里程格子：放大系数 1.5，所以更宽 */
```
> `flex: 1 1 230px` 三个值是 `flex-grow flex-shrink flex-basis`——**放大比例、缩小比例、基准尺寸**。把里程格子的放大比例设成 `1.5`，它就会**分到更多剩余空间**，比其它格子宽，形成视觉重点。

**格子间的金色分隔线**（用伪元素，不加标签）：
```css
.sl-fig + .sl-fig::before {     /* "+"是相邻兄弟选择器：每个挨着前一个格子的格子 */
  content: ""; position: absolute; left: 0;
  width: 1px; background: linear-gradient(transparent, rgba(212,175,55,.4), transparent);
}
```
> `.sl-fig + .sl-fig` 的 `+` 是**相邻兄弟选择器**，意思是"紧跟在另一个 `.sl-fig` 后面的 `.sl-fig`"——这样**第一个格子前面不会有分隔线**，只在格子之间出现，很巧妙。

**数字滚动**（机制同首页 3.2，用 ScrollTrigger 在进入视野时触发 `countUp`）：
```js
document.querySelectorAll(".sl-figures [data-target]").forEach(function (num) {
  ScrollTrigger.create({ trigger: num, start: "top 88%", once: true,
    onEnter: function () { countUp(num); } });   // 进入视野时跳数字，只跑一次
});
```

#### ④ 一句话讲 + 追问
> **一句话**："数字带用弹性盒布局，通过把'里程'格子的 `flex-grow` 设为 1.5 实现**不等分**、突出重点；`flex-wrap` 保证窄屏换行。数字用 GSAP 配合 ScrollTrigger，在滚入视野时从 0 递增到目标值，只触发一次。"

> **追问：`flex: 1.5 1 300px` 这三个数字分别是什么？**
> **答**："分别是 `flex-grow`（放大比例）、`flex-shrink`（缩小比例）、`flex-basis`（基准宽度）。放大比例 1.5 意味着在分配多余空间时，这个格子比放大比例为 1 的格子多拿 50%，所以更宽。"

### 4.5 图片兜底机制（工程亮点，强烈建议讲）

#### ① 效果
万一某张图片**加载失败**（路径错、文件缺失），不会显示难看的"破图"图标，而是：先尝试**换几个扩展名**（.jpg→.png 等）再加载；都失败就换成一块**带图名文字的渐变色块**占位，页面整体观感不崩。

#### ② 知识点
- JS 监听图片 `error` 事件
- `dataset` 记录已尝试路径
- DOM 操作：动态创建元素并 `replaceWith` 替换

#### ③ 代码逐行讲（`千年丝路.js`）
```js
function fallback(img) {
  // 1) 先尝试备用扩展名：.jpg 换 .png、再换 .jpg.png 等
  var cur = img.getAttribute("src");
  // ...（拼出候选路径，挑一个没试过的赋给 img.src 重试）...
  if (next) { img.src = next; return; }

  // 2) 所有路径都失败 → 造一个渐变色块顶替
  var ph = document.createElement("div");          // 新建一个 div
  ph.className = img.className;                     // 复制原图的 class（保持布局）
  ph.style.background = img.dataset.fallback        // 用 HTML 上 data-fallback 指定的渐变
      || "linear-gradient(135deg,#5a3a2a,#8b6a3a)"; // 没指定就用默认渐变
  ph.textContent = img.getAttribute("alt") || "";  // 把图片的 alt 文字写上去
  img.replaceWith(ph);                             // 用色块替换掉破图
}
// 给每张图绑定"加载失败"监听
document.querySelectorAll("img.sl-epoch__img, img.sl-slide__img").forEach(function (img) {
  img.addEventListener("error", function () { fallback(img); });
});
```
HTML 里每张图预先写好兜底渐变：
```html
<img class="sl-epoch__img" src="../images/张骞.png" alt="张骞出使西域"
     data-fallback="linear-gradient(150deg,#7a4a2c,#caa15e)">
```

#### ④ 一句话讲 + 追问
> **一句话**："我为所有壁画图做了**加载失败兜底**：监听图片的 `error` 事件，先自动尝试几个备用扩展名重新加载；若仍失败，就用 JS 动态创建一个**带图名文字的渐变色块**替换掉破图，保证即使资源缺失，页面排版和观感也不会崩坏。"

> **追问：为什么要做图片兜底？**
> **答**："真实项目中图片可能因路径错误、网络问题、文件丢失而加载失败，默认会显示难看的破图图标，破坏整体设计。做兜底是一种**健壮性（robustness）** 和**优雅降级**的工程思维——保证最坏情况下用户体验依然可接受。"

> **追问：`replaceWith` 和 `appendChild` 区别？**
> **答**："`appendChild` 是往里**追加**子元素；`replaceWith` 是把当前元素整个**替换**成新元素。这里要把破图替换成色块，所以用 `replaceWith`。"

## 第五章 · 光影敦煌 光影敦煌.html 逐块详解

> 这是一个**暗色画廊**页面，主题"黑白沉淀历史、触碰焕发新生"。类名统一用 `gy-` 前缀。JS 交互最丰富：黑白焕彩、拖动对比、热点切换。布局借助 Bootstrap 栅格。

### 5.1 暗色场景切换（特异性技巧）

#### ① 效果
全站默认是浅沙色背景，但**这一页整体翻成深黑色**画廊氛围。

#### ② 知识点
- CSS **选择器特异性（specificity）**
- 给 `<body>` 加专属类来"提权覆盖"

#### ③ 代码逐行讲
HTML 的 body 上挂了个专属类：
```html
<body class="gy-stage" data-reveal-threshold="0.1">
```
CSS 里用**两个选择器叠加**来提高优先级：
```css
body.gy-stage {                /* 注意：body 和 .gy-stage 连写，特异性更高 */
  background-color: #0d0b0a;   /* 稳稳盖过全局的浅沙底 */
  color: #a89f89;
}
```
> 为什么不直接写 `.gy-stage`？因为全局 `dunhuang.css` 里 `body { background: var(--sand) }` 已经给 body 上了色。要可靠覆盖它，需要**更高的特异性**——把标签 `body` 和类 `.gy-stage` 连写（`body.gy-stage`），优先级就高于单独的 `body`，从而稳定生效。

#### ④ 一句话讲 + 追问
> **一句话**："这一页通过给 `<body>` 加 `gy-stage` 类切换成暗色场景。我用 `body.gy-stage` 这种**标签+类连写**的选择器提高特异性，确保稳定覆盖全局的浅色背景。"

> **追问（进阶必考）：什么是 CSS 特异性？为什么 `body.gy-stage` 比 `body` 优先级高？**
> **答**："特异性是浏览器决定'多个规则冲突时听谁的'的打分机制。简单说：**ID > 类 > 标签**。`body` 只有一个标签（1 分），`body.gy-stage` 是一个标签 + 一个类（1+10 分），分更高所以胜出。这是可靠覆盖全局样式的常用手法，比滥用 `!important` 更优雅。"

### 5.2 飞天写真 · Bootstrap 栅格 + 黑白焕彩

#### ① 效果
四张飞天画卡，整齐排成**两列网格**（手机上变一列）。每张默认是**黑白沉睡**状态，鼠标悬停或触摸时**变为彩色并轻微放大**，标题也变亮。

#### ② 知识点
- **Bootstrap 栅格系统**（`row` / `col`）
- CSS `filter: grayscale()` 黑白→彩色
- `:hover` / `:focus-within` + `transition`
- `aspect-ratio` 固定卡片比例

#### ③ 代码逐行讲

**Bootstrap 栅格做响应式两列**（HTML）：
```html
<div class="row g-4">                    <!-- row=一行，g-4=格子间距 -->
  <div class="col-12 col-md-6">          <!-- 手机占满12列(整行)，中屏以上占6列(半行) -->
    <article class="gy-card" tabindex="0">...</article>
  </div>
  <!-- 共四个 col-12 col-md-6，于是中屏两列、手机一列 -->
</div>
```
> Bootstrap 把每行分成 **12 列**。`col-12` = 占满 12 格（整行，手机上一张一行）；`col-md-6` = 中等屏幕(md)以上占 6 格（半行，两张并排）。**一套类名搞定响应式布局**，这就是栅格系统的威力。`tabindex="0"` 让卡片能被键盘 Tab 聚焦。

**黑白焕彩**（CSS 核心）：
```css
.gy-img {
  filter: grayscale(100%) brightness(0.7) contrast(1.08);  /* 默认：100%灰度+压暗 */
  transition: filter 0.6s cubic-bezier(0.25,1,0.5,1), transform 0.6s ease;
}
.gy-card:hover .gy-img,
.gy-card:focus-within .gy-img {
  filter: grayscale(0) brightness(1) contrast(1);  /* 悬停/聚焦：灰度归零=恢复彩色 */
  transform: scale(1.05);                          /* 轻微放大 */
}
```
> `filter: grayscale(100%)` 把图片变成**100% 黑白**；悬停时设为 `grayscale(0)` 恢复全彩。配合 0.6 秒 `transition`，就有了"沉睡的色彩被唤醒"的主题表达。`cubic-bezier(...)` 是自定义的缓动曲线，让过渡更有质感。

#### ④ 一句话讲 + 追问
> **一句话**："飞天写真用 **Bootstrap 栅格** 的 `col-12 col-md-6` 实现'手机一列、桌面两列'的响应式布局。每张图默认用 `filter: grayscale(100%)` 变黑白，悬停或键盘聚焦时灰度归零恢复彩色并放大，呼应'黑白沉淀、触碰焕新'的主题。"

> **追问：Bootstrap 栅格的 `col-12 col-md-6` 是什么意思？**
> **答**："Bootstrap 把一行分成 12 等份。`col-12` 表示占满 12 份（独占一行），`col-md-6` 表示在中等屏幕（≥768px）及以上占 6 份（半行）。两个写一起，就实现了小屏一列、大屏两列的**响应式**，不用自己写媒体查询。"

> **追问：`grayscale()` 滤镜的原理？**
> **答**："`filter: grayscale(100%)` 是 CSS 滤镜函数，按比例把图片去色。100% 是完全黑白，0% 是原彩色。它由浏览器在渲染层处理，性能好、只需一个属性。"

### 5.3 敦煌色谱 · 矿物色卡

#### ① 效果
六块**矿物颜料色卡**（青金石、朱砂、石绿等），默认**去色压暗**像在幽暗洞窟里；悬停时色块**饱和度恢复、上浮**，下方说明文字**淡入显现**。

#### ② 知识点
- CSS `filter: saturate()` 控制饱和度
- `radial-gradient` 做矿物微光质感
- 悬停时 `opacity` 控制说明显隐
- 触屏 `@media (hover: none)` 兼容

#### ③ 代码逐行讲
颜色用 HTML 内联变量传入：
```html
<article class="gy-pig" tabindex="0" style="--c:#1f3a93">  <!-- 把色值存进 --c 变量 -->
  <div class="gy-pig-chip"></div>
  <h4 class="gy-pig-name">青金石</h4>
  <p class="gy-pig-desc">来自葱岭以西、经丝路东来的珍贵之蓝……</p>
</article>
```
```css
.gy-pig-chip {
  background: var(--c, #555);            /* 用 HTML 传进来的 --c 当颜色 */
  filter: saturate(0.22) brightness(0.55);  /* 默认：低饱和+压暗=沉睡 */
  transition: filter 0.6s, transform 0.5s;
}
.gy-pig:hover .gy-pig-chip {
  filter: saturate(1) brightness(1);    /* 悬停：恢复饱和+提亮=苏醒 */
  transform: translateY(-5px);          /* 上浮 5px */
}
.gy-pig-desc { opacity: 0; transition: opacity 0.45s; }   /* 说明默认隐藏 */
.gy-pig:hover .gy-pig-desc { opacity: 0.9; }              /* 悬停淡入 */

@media (hover: none) {            /* 触屏设备没有悬停 */
  .gy-pig-desc { opacity: 0.9; }  /* 直接常显，保证手机能看到说明 */
}
```
> 注意 `var(--c, #555)` 的第二个值 `#555` 是**默认色**——万一 HTML 没传 `--c`，就用灰色兜底。触屏没有 hover，所以专门用 `@media (hover: none)` 让说明文字常显。

#### ④ 一句话讲 + 追问
> **一句话**："色谱卡片默认用 `filter: saturate(0.22) brightness(0.55)` 去色压暗模拟'洞窟幽暗'，悬停恢复饱和并上浮、说明文字淡入。颜色通过 HTML 内联的 `--c` 变量传入，便于每块单独配色。还用 `@media (hover: none)` 为触屏设备让说明常显。"

> **追问：颜色为什么用 `style="--c:..."` 传进来，而不是写死在 CSS？**
> **答**："六块色卡结构完全一样，只有颜色不同。把颜色作为**变量从 HTML 传入**，CSS 只写一套通用规则即可，避免给六块各写一遍样式，**结构和数据分离**，更易维护。"

### 5.4 数字复原 · clip-path 拖动对比（JS 交互精华）

#### ① 效果
一张壁画的"**残损/褪色**"版和"**复原/焕彩**"版**叠在一起**，中间有一条带圆钮的**分割线**。拖动滑块，左右揭示——看壁画如何从残损"复原"。

#### ② 知识点
- CSS `clip-path: inset()` 裁剪揭示
- 用 CSS 变量 `--pos` 联动裁剪位置和分割线
- **透明 range 滑块**覆盖在上层接收拖动
- JS 把滑块值写入 CSS 变量

#### ③ 代码逐行讲

**两层图叠放，上层用 clip-path 裁剪**：
```css
.gy-compare { position: relative; }            /* 定位上下文 */
.gy-cmp-layer { position: absolute; inset: 0; } /* 两层都铺满、叠在一起 */
.gy-cmp-after {                                /* 复原层（在上） */
  clip-path: inset(0 calc(100% - var(--pos, 50%)) 0 0);
  /* 用 --pos 控制：右侧裁掉 (100%-pos)，即只露出左边 pos 这部分复原图 */
}
.gy-cmp-line { left: var(--pos, 50%); }        /* 分割线也跟着 --pos 走 */
.gy-cmp-knob { left: var(--pos, 50%); }        /* 圆钮同理 */
```
> `clip-path: inset(上 右 下 左)` 是**裁剪**：把元素四边各裁掉多少。这里只裁右边，裁掉量 = `100% - --pos`。所以 `--pos` 越大，复原图露出越多。分割线、圆钮的 `left` 也都绑 `--pos`，于是它们**永远对齐**。

**透明滑块接收拖动**：
```css
.gy-cmp-range {
  position: absolute; inset: 0; width: 100%; height: 100%;
  opacity: 0;            /* 完全透明：看不见，但能拖 */
  cursor: ew-resize;     /* 鼠标变成左右拖动样式 */
}
```
**JS 把滑块的值写进 `--pos`**（`光影敦煌.js`）：
```js
document.querySelectorAll(".gy-compare").forEach(function (cmp) {
  var range = cmp.querySelector(".gy-cmp-range");
  function apply() { cmp.style.setProperty("--pos", range.value + "%"); }  // 滑块值→--pos
  range.addEventListener("input", apply);  // 每次拖动都更新
  apply();                                 // 初始化一次
});
```
> 一个铺满全区、完全透明的 `<input type="range">` 盖在最上层负责接收拖动；JS 监听它的 `input` 事件，把当前值（0~100）写进 CSS 变量 `--pos`，于是裁剪、分割线、圆钮全部联动。**用原生 range 还自带键盘可访问性**（左右方向键也能调）。

#### ④ 一句话讲 + 追问
> **一句话**："数字复原是个拖动对比组件：残损图和复原图叠放，上层复原图用 `clip-path: inset()` 配合 CSS 变量 `--pos` 控制揭示比例。一个全屏透明的原生 range 滑块接收拖动，JS 把它的值写入 `--pos`，让裁剪线、分割线、圆钮三者联动。"

> **追问（必考）：拖动揭示是怎么实现的？**
> **答**："核心是 `clip-path: inset()` 裁剪。把复原图盖在残损图上，用 `clip-path` 裁掉它右边一部分，裁多少由 CSS 变量 `--pos` 决定。一个透明的 range 滑块叠在最上层接收拖动，JS 监听它的 `input` 事件把数值写进 `--pos`，从而实时控制揭示宽度，同时分割线和圆钮也绑定 `--pos` 保持对齐。"

> **追问：为什么用透明的原生 `<input type="range">` 而不是自己用 JS 监听鼠标？**
> **答**："用原生 range 有三个好处：**自带拖动逻辑**（不用手写 mousedown/mousemove/mouseup）、**自带键盘可访问性**（方向键可调）、**自带触屏支持**。把它设透明盖在上层，既复用了浏览器能力又不破坏视觉，是省力又健壮的做法。"

### 5.5 壁画探微 · 热点交互（JS 切换说明）

#### ① 效果
一张飞天壁画上散布几个**会脉冲闪动的光点**。点击某个光点，它**高亮**，下方面板**切换显示该处的细节说明**（祥云、天女散花、璎珞华饰等）。

#### ② 知识点
- 绝对定位 + 内联 `left/top` 摆放热点
- CSS `@keyframes` 脉冲动画
- HTML `data-*` 存放每个热点的标题和说明
- JS 点击切换：清高亮 → 高亮当前 → 换文字

#### ③ 代码逐行讲

**热点定位与脉冲**（HTML + CSS）：
```html
<button class="gy-hs-dot" style="left:31%;top:11%"
        data-title="祥云缭绕" data-desc="满壁流转的青绿与赭金祥云……">+</button>
```
```css
.gy-hs-dot { position: absolute; transform: translate(-50%,-50%); animation: gy-pulse 2.4s infinite; }
@keyframes gy-pulse {     /* 脉冲：外圈光晕一胀一消，循环 */
  0%,100% { box-shadow: 0 0 0 0 rgba(203,182,130,.5); }
  50%     { box-shadow: 0 0 0 9px rgba(203,182,130,0); }
}
.gy-hs-dot.active { background: var(--gold); animation: none; }  /* 选中时停止脉冲、变金 */
```
> 每个光点用内联 `left/top` 百分比**精确钉在壁画对应位置**；`@keyframes gy-pulse` 让 `box-shadow` 外圈一圈圈扩散又消失，形成**呼吸脉冲**，提示"这里可以点"。

**点击切换说明**（`光影敦煌.js`）：
```js
function showDetail(dot) {
  for (var i = 0; i < dots.length; i++) dots[i].classList.remove("active"); // 1.先清掉所有高亮
  dot.classList.add("active");                                              // 2.高亮被点的这个
  panel.innerHTML = "<h4>" + dot.dataset.title + "</h4><p>" + dot.dataset.desc + "</p>";
  // 3.把这个光点的 data-title / data-desc 写进下方面板
}
for (var i = 0; i < dots.length; i++) {
  dots[i].onclick = function () { showDetail(this); };  // this=被点的那个光点
}
if (dots.length) showDetail(dots[0]);  // 默认先显示第一处
```
> 每个光点的标题和说明都存在它自己的 `data-title`、`data-desc` 上。点击时：先把所有光点的 `active` 高亮清掉，再给当前点加上，最后用它的 `dataset` 数据**刷新下方说明面板**。`showDetail(this)` 里的 `this` 就是被点击的那个光点。

#### ④ 一句话讲 + 追问
> **一句话**："壁画探微是个热点交互。每个光点用绝对定位的百分比钉在壁画对应部位，用 `@keyframes` 做脉冲呼吸提示可点。说明文字存在各自的 `data-*` 属性里，点击时 JS 切换高亮并用该点的 `dataset` 数据刷新下方说明面板。"

> **追问：每个热点的文字是怎么和光点关联的？**
> **答**："存在 HTML 的自定义属性 `data-title` 和 `data-desc` 上，**数据跟着元素走**。点击时 JS 通过 `dot.dataset.title` 读出来填进面板。这样新增一个热点只要加一个带 `data-*` 的按钮即可，无需改 JS，**易扩展**。"

> **追问：`showDetail(this)` 里的 `this` 是什么？**
> **答**："在事件处理函数里，`this` 指向**触发事件的那个元素**，也就是被点击的光点。所以同一个函数能服务所有光点，谁被点就处理谁。"

### 5.6 滚动出场（本页的 IntersectionObserver）

#### ① 效果
四个篇章（飞天写真、敦煌色谱、数字复原、壁画探微）随滚动**依次淡入上浮**出场。

#### ② 知识点
与全局 `脚本.js` 同款 **IntersectionObserver**，但本页 `光影敦煌.js` 单独写了一份，观察 `.gy-chapter.reveal`。

#### ③ 代码逐行讲
```js
var io = new IntersectionObserver(function (entries) {
  entries.forEach(function (e) {
    if (e.isIntersecting) {        // 篇章进入视野
      e.target.classList.add("in"); // 加 in 触发 CSS 淡入
      io.unobserve(e.target);       // 只播一次
    }
  });
}, { threshold: threshold });
document.querySelectorAll(".gy-chapter.reveal").forEach(function (t) { io.observe(t); });
```
配套 CSS：
```css
.gy-chapter.reveal { opacity: 0; transform: translateY(24px); transition: opacity .9s, transform .9s; }
.gy-chapter.reveal.in { opacity: 1; transform: none; }
```

#### ④ 一句话讲 + 追问
> **一句话**："本页四个篇章用 IntersectionObserver 实现滚动出场——滚进视野才淡入上浮，每个只触发一次。原理和全局滚动浮现一致，这里针对本页的 `.gy-chapter` 单独观察。"

> **追问：这页已经引了全局 `脚本.js`，为什么还单独写一份观察器？**
> **答**："全局 `脚本.js` 观察的是 `.reveal` 类，而本页篇章用的是 `.gy-chapter.reveal` 并配了本页专属的淡入样式与阈值（`data-reveal-threshold="0.1"`）。单独写一份能**针对本页定制**触发灵敏度和动画，互不干扰。"

### 6.1 整体技术栈类

**Q：请整体介绍一下你这个网站用了哪些技术？**
A："网站基于最基础的 **HTML + CSS + JavaScript** 三件套：HTML 负责结构、CSS 负责样式、JS 负责交互。在此之上，布局和轮播借助了 **Bootstrap** 这个 CSS 框架提高效率；复杂的滚动动画用了业界标准的 **GSAP** 动画引擎及其 **ScrollTrigger** 插件。整体采用'结构、表现、行为'三层分离的思想，并通过 CSS 变量统一设计风格、用类名前缀做命名空间隔离。"

**Q：为什么有的动画用纯 CSS，有的用 GSAP？怎么取舍的？**
A："**简单的状态切换（如 hover 变色、菜单展开）我用纯 CSS**，因为性能好、代码简洁、不依赖 JS。**需要跟随滚动进度、精确编排时序的复杂动画（如时间轴生长、横向长卷、数字递增）我用 GSAP**，因为这类效果纯 CSS 很难做或做不了，GSAP 的 ScrollTrigger 能把动画和滚动绑定，控制力强。原则是'能用 CSS 就不用 JS，该用 GSAP 时果断用'。"

**Q：CSS 和 JS 的分工你是怎么理解的？**
A："CSS 负责**静态外观和由用户即时操作（hover/focus）触发的简单过渡**；JS 负责**需要逻辑判断、读写数据、响应复杂事件（滚动、点击切换、拖动）的行为**。比如掌灯夜探纯靠 CSS hover 就够了，但壁画热点要根据点的是哪个来切换不同文字，必须用 JS。"

### 6.2 响应式与兼容类

**Q：你的网站怎么适配手机？**
A："多管齐下：① 用 **Bootstrap 栅格**的 `col-12 col-md-6` 做'手机一列、桌面多列'；② 用 **媒体查询 `@media`** 在窄屏调整布局，比如导航折叠成汉堡菜单、时间轴从居中改成靠左堆叠；③ 用 **`clamp()`** 让字号随屏幕自适应；④ 用 **`vw/vh` 相对单位**让间距按比例缩放；⑤ 针对触屏用 `@media (hover: none)` 处理没有悬停的情况。"

**Q：视差背景在手机上有问题，你怎么处理的？**
A："`background-attachment: fixed` 在 iOS Safari 等移动浏览器上会失效或卡顿。我用 `@media (hover: none), (max-width: 768px)` 检测触屏/窄屏，把它**降级**为 `scroll`（背景随页面正常滚动），牺牲视差换取流畅，这是**优雅降级**的思路。"

### 6.3 性能与健壮性类（容易出彩）

**Q：你做了哪些性能优化？**
A："① 滚动监听用 **IntersectionObserver** 而非高频的 `scroll` 事件，由浏览器底层优化；② 出场动画**只触发一次**（`unobserve` / `once`），不重复计算；③ 动画尽量用 `transform` 和 `opacity`（这两个属性由 GPU 合成、不引起重排，性能最好）；④ 图片加 `loading="lazy"` 懒加载；⑤ 简单交互用纯 CSS，不占用 JS 线程。"

**Q：如果图片加载失败、或者 GSAP 没加载出来，页面会崩吗？**
A："不会，我做了两层兜底。**图片失败**：监听 `error` 事件，自动尝试备用扩展名，再不行就用渐变色块占位。**GSAP/动画失败**：每段动画脚本都有'无 GSAP 或用户开了减弱动效'的兜底分支，直接把元素设为最终可见状态——内容**始终可见**，动画只是锦上添花。这是**渐进增强 / 优雅降级**的工程思维。"

**Q：你考虑无障碍（Accessibility）了吗？**
A："考虑了。① 导航按钮加了 `aria-label`、`aria-expanded` 给读屏软件；② 交互元素同时支持 `:hover` 和 `:focus-within`，**键盘也能操作**；③ 卡片加 `tabindex="0"` 可被 Tab 聚焦；④ 用 `prefers-reduced-motion` 媒体查询，为晕动症用户**自动关闭动画**；⑤ 拖动对比用原生 range，自带键盘可访问性；⑥ 图片都写了 `alt` 替代文字。"

### 6.4 架构与可维护性类

**Q：三个页面的样式怎么组织的？怎么避免冲突？**
A："分三层：**Bootstrap**（第三方基础）→ **dunhuang.css**（全站共用的变量、导航、页脚、通用动效）→ **各页专属 CSS**。靠 CSS 加载顺序保证'后者覆盖前者'。为避免冲突，每页类名加专属前缀（`sl-`/`gy-`）做**命名空间隔离**，公共部分（如页脚）则**收口到全局**避免重复。"

**Q：CSS 变量（设计令牌）给你带来了什么好处？**
A："统一和可维护。全站主色、字体、圆角都定义在 `:root`，用 `var()` 引用。想调整风格只改一处，三个页面同步生效，杜绝了'改色改几十处还漏改'的问题，也让代码语义更清晰（`var(--red)` 比 `#8b2626` 易读）。"

### 6.5 万一被问到不会的，怎么稳住？

- **别慌，别编**。可以说："这一块我主要实现了……（讲你会的部分），关于您问的这个细节我理解是……，如果有不准确的地方请您指正。"
- **把话题引导到你最熟的区块**（比如时光卷轴、数字复原），主动展开。
- **承认 + 给方向**永远比**硬编错答案**好："这个具体参数我需要回看代码确认，但它的作用是控制……"

## 附录 · 名词小词典

> 答辩时若被问到某个词，可快速翻这里。按出现频率排序。

| 名词 | 一句话解释 |
|---|---|
| **HTML** | 超文本标记语言，定义网页**结构和内容**（有什么） |
| **CSS** | 层叠样式表，定义网页**外观**（长什么样） |
| **JavaScript（JS）** | 脚本语言，定义网页**交互行为**（点了/滚了会怎样） |
| **标签 tag** | HTML 的基本单位，如 `<p>`、`<img>`，成对出现 |
| **class 类名** | 给元素贴的"名字标签"，供 CSS/JS 精确选中 |
| **选择器** | CSS 中"选中谁"的写法，如 `.site-nav` |
| **CSS 变量 / 设计令牌** | 用 `--名字` 定义、`var(--名字)` 引用的可复用值，统一管理颜色字体等 |
| **特异性 specificity** | 多条 CSS 冲突时谁说了算的优先级打分（ID>类>标签） |
| **弹性盒 Flexbox** | 一维（一行或一列）的布局方式，`display:flex` |
| **栅格 Grid** | 二维（行+列）的布局方式，`display:grid` |
| **flex 比例** | `flex:1 1 230px` = 放大比例、缩小比例、基准尺寸 |
| **定位 position** | `fixed` 钉在屏幕、`absolute` 相对父级精确摆放、`relative` 定位参照 |
| **z-index** | 元素堆叠层级，越大越靠上层 |
| **transition 过渡** | 样式变化时的平滑动画（需触发，如 hover） |
| **@keyframes 关键帧** | 定义可自动播放的多帧复杂动画 |
| **animation-delay** | 动画延迟开始的时间，用于"错峰入场" |
| **transform 变换** | 移动/缩放/旋转元素（`translate`/`scale`/`rotate`），性能好 |
| **filter 滤镜** | 给元素加视觉效果：`grayscale` 黑白、`sepia` 泛黄、`saturate` 饱和度、`blur` 模糊 |
| **clip-path 裁剪** | 按形状裁剪元素，`inset()` 是矩形内缩裁剪 |
| **mix-blend-mode** | 元素与背景的混合模式，`multiply`(正片叠底) 可去白底融合 |
| **mask-image 蒙版** | 用渐变控制元素哪里显示哪里透明，做羽化淡出 |
| **媒体查询 @media** | 按屏幕宽度/特性应用不同样式，实现响应式 |
| **clamp(min,理想,max)** | 自适应取值，常用于字号，自动夹在最小最大之间 |
| **vw / vh** | 相对视口宽/高的单位，1vw=屏幕宽的1% |
| **响应式 responsive** | 一套页面自适应手机/平板/电脑不同屏幕 |
| **视差 parallax** | 背景与前景滚动速度不同，制造景深错觉 |
| **background-attachment:fixed** | 背景相对窗口固定，实现最简视差 |
| **Bootstrap** | 流行的 CSS 框架，提供栅格、轮播等现成组件 |
| **栅格 12 列** | Bootstrap 把每行分 12 份，`col-md-6`=中屏占半行 |
| **Carousel 轮播** | Bootstrap 的图片轮播组件，靠 `data-bs-*` 驱动 |
| **data-* 属性** | HTML 自定义数据属性，JS 用 `元素.dataset.xxx` 读取 |
| **GSAP** | 业界标准的 JS 动画引擎，控制力强 |
| **ScrollTrigger** | GSAP 插件，让动画跟随滚动进度 |
| **gsap.to / from** | 动画到指定状态 / 从指定状态进场 |
| **gsap.timeline** | 把多个动画串成一条可编排的时间线 |
| **scrub** | 把动画进度绑定滚动条，"边滚边播、可倒退" |
| **pin** | ScrollTrigger 把区域"钉"在屏幕上不随页面滚 |
| **IntersectionObserver** | 浏览器原生"是否进入视野"观察器，高效，做滚动浮现 |
| **addEventListener** | JS 给元素绑定事件监听（如 click、error、input） |
| **classList.toggle/add/remove** | JS 增删切换元素的 class，从而改变样式 |
| **this** | 事件处理函数里指向"触发事件的那个元素" |
| **prefers-reduced-motion** | 检测用户"减少动态效果"偏好，用于关闭动画做无障碍 |
| **aria-* 属性** | 无障碍属性，给读屏软件提供信息 |
| **优雅降级 / 渐进增强** | 保证基础功能在最差环境也能用，高级效果锦上添花 |
| **命名空间 / 类名前缀** | 用 `sl-`/`gy-` 等前缀隔离样式，防止冲突 |
