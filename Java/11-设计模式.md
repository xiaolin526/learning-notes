# Java 常用设计模式

**设计模式（Design Patterns）** 是前人总结的、解决特定问题的**代码套路**。学会它们能让你的代码更优雅、更易维护，也是面试和阅读源码的必备基础。

## 一、设计模式分类

GoF（Gang of Four）将 23 种经典设计模式分为三大类：

| 类型      | 关注点              | 典型模式                       |
| ------- | ---------------- | -------------------------- |
| **创建型** | 怎么创建对象           | 单例、工厂、建造者、原型              |
| **结构型** | 类和对象怎么组合         | 适配器、装饰器、代理、外观              |
| **行为型** | 对象之间怎么交互、分配职责    | 策略、观察者、模板方法、责任链            |

本篇重点讲**最常用的 9 个**。

## 二、六大设计原则（SOLID）

写好代码的指导思想：

| 原则                                   | 简单理解                  |
| ------------------------------------ | --------------------- |
| **S** 单一职责（Single Responsibility）    | 一个类只做一件事              |
| **O** 开闭原则（Open-Closed）              | 对扩展开放，对修改关闭           |
| **L** 里氏替换（Liskov Substitution）      | 子类可以替换父类，不破坏行为        |
| **I** 接口隔离（Interface Segregation）    | 接口要小而专，别一个接口塞一堆方法     |
| **D** 依赖倒置（Dependency Inversion）     | 依赖抽象（接口），不依赖具体实现      |
| 迪米特原则（Law of Demeter）                | 只和"朋友"通信，减少耦合         |

# 创建型模式

## 一、单例模式（Singleton）⭐⭐⭐⭐⭐

**保证一个类只有一个实例，并提供全局访问点。**

### 1. 饿汉式（推荐，简单线程安全）

```java
public class Singleton {
    // 类加载时就创建（"饿"——一上来就创建好）
    private static final Singleton INSTANCE = new Singleton();

    private Singleton() { }   // 构造方法私有，外部不能 new

    public static Singleton getInstance() {
        return INSTANCE;
    }
}
```

### 2. 双重检查锁（懒汉式，按需创建）

```java
public class Singleton {
    // volatile 防止指令重排
    private static volatile Singleton instance;

    private Singleton() { }

    public static Singleton getInstance() {
        if (instance == null) {                  // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {          // 第二次检查
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 3. 静态内部类（推荐，优雅）

```java
public class Singleton {
    private Singleton() { }

    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;   // 第一次调用才加载内部类，懒加载又线程安全
    }
}
```

### 4. 枚举单例（最优雅，最安全）

```java
public enum Singleton {
    INSTANCE;

    public void doSomething() {
        System.out.println("干活");
    }
}

// 使用
Singleton.INSTANCE.doSomething();
```

> 💡 **《Effective Java》作者 Joshua Bloch 推荐枚举单例**——天然防反射、防序列化破坏。

### 应用场景

数据库连接池、配置管理器、日志对象、Spring 的 Bean（默认）。

## 二、工厂模式（Factory）⭐⭐⭐⭐

**把对象的创建逻辑封装起来**，调用方不直接 new。

### 简单工厂

```java
// 产品接口
interface Shape {
    void draw();
}

class Circle implements Shape {
    public void draw() { System.out.println("画圆"); }
}

class Square implements Shape {
    public void draw() { System.out.println("画方"); }
}

// 工厂
class ShapeFactory {
    public static Shape create(String type) {
        return switch (type) {
            case "circle" -> new Circle();
            case "square" -> new Square();
            default -> throw new IllegalArgumentException("未知类型");
        };
    }
}

// 使用
Shape s = ShapeFactory.create("circle");
s.draw();
```

### 工厂方法

每种产品对应一个工厂，便于扩展：

```java
interface ShapeFactory {
    Shape create();
}

class CircleFactory implements ShapeFactory {
    public Shape create() { return new Circle(); }
}

class SquareFactory implements ShapeFactory {
    public Shape create() { return new Square(); }
}
```

### 应用场景

`Calendar.getInstance()`、`LoggerFactory.getLogger()`、Spring 的 BeanFactory。

## 三、建造者模式（Builder）⭐⭐⭐⭐

**链式调用构造复杂对象**，避免构造方法参数过多。

```java
public class User {
    private String name;
    private int age;
    private String email;
    private String phone;

    private User() { }

    public static Builder builder() {
        return new Builder();
    }

    public static class Builder {
        private final User user = new User();

        public Builder name(String name)   { user.name = name; return this; }
        public Builder age(int age)        { user.age = age; return this; }
        public Builder email(String email) { user.email = email; return this; }
        public Builder phone(String phone) { user.phone = phone; return this; }

        public User build() {
            // 这里可以做参数校验
            return user;
        }
    }
}

// 使用 —— 链式调用，可读性极高
User u = User.builder()
    .name("小林")
    .age(18)
    .email("xiaolin@example.com")
    .build();
```

### 应用场景

`StringBuilder`、`StringBuffer`、Lombok 的 `@Builder` 注解、`OkHttpClient.Builder`。

# 结构型模式

## 四、代理模式（Proxy）⭐⭐⭐⭐⭐

**为另一个对象提供一个代替品**，在不修改原对象的情况下增强功能。

### 静态代理

```java
interface UserService {
    void save();
}

class UserServiceImpl implements UserService {
    public void save() {
        System.out.println("保存用户");
    }
}

// 代理类：在原方法前后加入日志
class UserServiceProxy implements UserService {
    private UserService target;

    public UserServiceProxy(UserService target) {
        this.target = target;
    }

    public void save() {
        System.out.println("[日志] 方法开始");
        target.save();
        System.out.println("[日志] 方法结束");
    }
}

// 使用
UserService service = new UserServiceProxy(new UserServiceImpl());
service.save();
```

### 动态代理（JDK Proxy）

不用为每个类写代理类，运行时动态生成：

```java
import java.lang.reflect.*;

UserService target = new UserServiceImpl();

UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxy1, method, args) -> {
        System.out.println("[日志] 方法开始：" + method.getName());
        Object result = method.invoke(target, args);
        System.out.println("[日志] 方法结束");
        return result;
    }
);

proxy.save();
```

### 应用场景

- **Spring AOP**（事务、日志、权限）
- **MyBatis 的 Mapper 接口**（你写接口，框架自动生成实现）
- **RPC 框架**（远程调用看起来像本地调用）

> 💡 **代理模式是 Spring 框架的核心。** 你以后看 Spring 源码，到处都是动态代理。

## 五、装饰器模式（Decorator）⭐⭐⭐⭐

**动态给对象添加功能**，比继承更灵活。

```java
// 接口
interface Coffee {
    String getDescription();
    double getCost();
}

// 基础咖啡
class SimpleCoffee implements Coffee {
    public String getDescription() { return "咖啡"; }
    public double getCost() { return 10.0; }
}

// 装饰器基类
abstract class CoffeeDecorator implements Coffee {
    protected Coffee coffee;
    public CoffeeDecorator(Coffee coffee) { this.coffee = coffee; }
}

// 装饰器：加奶
class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) { super(coffee); }
    public String getDescription() { return coffee.getDescription() + " + 奶"; }
    public double getCost() { return coffee.getCost() + 2.0; }
}

// 装饰器：加糖
class SugarDecorator extends CoffeeDecorator {
    public SugarDecorator(Coffee coffee) { super(coffee); }
    public String getDescription() { return coffee.getDescription() + " + 糖"; }
    public double getCost() { return coffee.getCost() + 1.0; }
}

// 使用 —— 想加几层加几层
Coffee c = new SugarDecorator(new MilkDecorator(new SimpleCoffee()));
System.out.println(c.getDescription() + " ￥" + c.getCost());
// 咖啡 + 奶 + 糖 ￥13.0
```

### 应用场景

Java IO 流就是经典的装饰器模式：
```java
new BufferedReader(            // 装饰：加缓冲
    new InputStreamReader(     // 装饰：字节→字符
        new FileInputStream("test.txt")));   // 基础流
```

## 六、适配器模式（Adapter）⭐⭐⭐

**让接口不兼容的两个类能一起工作**，类似"转接头"。

```java
// 旧接口
interface OldCharger {
    void chargeWithRound();
}

// 新设备需要这个接口
interface NewCharger {
    void chargeWithUsb();
}

// 旧充电器实现
class OldChargerImpl implements OldCharger {
    public void chargeWithRound() { System.out.println("圆口充电"); }
}

// 适配器：把旧充电器适配成新接口
class ChargerAdapter implements NewCharger {
    private OldCharger oldCharger;
    public ChargerAdapter(OldCharger oldCharger) { this.oldCharger = oldCharger; }

    public void chargeWithUsb() {
        System.out.println("[适配中]");
        oldCharger.chargeWithRound();
    }
}

// 使用
NewCharger charger = new ChargerAdapter(new OldChargerImpl());
charger.chargeWithUsb();
```

### 应用场景

- `Arrays.asList()` 把数组适配成 List
- SLF4J 适配各种日志框架（log4j、logback）

# 行为型模式

## 七、策略模式（Strategy）⭐⭐⭐⭐⭐

**把算法封装成独立的类，运行时可以互换。** 替代大堆 if-else 的神器。

```java
// 策略接口
interface PayStrategy {
    void pay(double amount);
}

class AlipayStrategy implements PayStrategy {
    public void pay(double amount) {
        System.out.println("支付宝支付 ¥" + amount);
    }
}

class WechatStrategy implements PayStrategy {
    public void pay(double amount) {
        System.out.println("微信支付 ¥" + amount);
    }
}

class CreditCardStrategy implements PayStrategy {
    public void pay(double amount) {
        System.out.println("信用卡支付 ¥" + amount);
    }
}

// 使用方
class PaymentContext {
    private PayStrategy strategy;
    public void setStrategy(PayStrategy strategy) { this.strategy = strategy; }
    public void pay(double amount) { strategy.pay(amount); }
}

// 使用
PaymentContext ctx = new PaymentContext();
ctx.setStrategy(new AlipayStrategy());
ctx.pay(100);
ctx.setStrategy(new WechatStrategy());
ctx.pay(200);
```

### 反例对比

```java
// ❌ 用 if-else 的写法
public void pay(String type, double amount) {
    if (type.equals("alipay")) {
        System.out.println("支付宝支付");
    } else if (type.equals("wechat")) {
        System.out.println("微信支付");
    } else if (type.equals("credit")) {
        System.out.println("信用卡支付");
    }
    // 加新方式就要改这里 → 违反开闭原则
}
```

### 应用场景

支付方式选择、排序算法、优惠规则、`Comparator`。

## 八、观察者模式（Observer）⭐⭐⭐⭐

**一个对象状态变化时，自动通知所有依赖它的对象。** 又叫"发布-订阅"。

```java
import java.util.ArrayList;
import java.util.List;

// 观察者接口
interface Observer {
    void update(String message);
}

// 被观察者
class Subject {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer o) { observers.add(o); }
    public void removeObserver(Observer o) { observers.remove(o); }

    public void notifyAll(String message) {
        for (Observer o : observers) {
            o.update(message);
        }
    }
}

// 具体观察者
class EmailSubscriber implements Observer {
    private String name;
    public EmailSubscriber(String name) { this.name = name; }
    public void update(String message) {
        System.out.println(name + " 收到邮件：" + message);
    }
}

// 使用
Subject blog = new Subject();
blog.addObserver(new EmailSubscriber("Alice"));
blog.addObserver(new EmailSubscriber("Bob"));
blog.notifyAll("有新文章发布！");
```

### 应用场景

- **事件监听**（按钮点击、消息推送）
- **MQ 消息订阅**（Kafka、RabbitMQ）
- Spring 的 `ApplicationEvent`

## 九、模板方法模式（Template Method）⭐⭐⭐

**父类定义算法骨架，子类填充具体步骤。**

```java
abstract class CookingTemplate {
    // 模板方法（final 防止子类重写整个流程）
    public final void cook() {
        prepare();
        addIngredients();
        heat();
        serve();
    }

    protected void prepare() {
        System.out.println("准备厨具");   // 通用步骤
    }

    protected abstract void addIngredients();   // 子类必须实现
    protected abstract void heat();

    protected void serve() {
        System.out.println("装盘上桌");   // 通用步骤
    }
}

class FriedRice extends CookingTemplate {
    protected void addIngredients() {
        System.out.println("加米饭、鸡蛋、葱花");
    }
    protected void heat() {
        System.out.println("热锅炒 3 分钟");
    }
}

class Noodle extends CookingTemplate {
    protected void addIngredients() {
        System.out.println("加面条、汤底");
    }
    protected void heat() {
        System.out.println("煮 5 分钟");
    }
}

// 使用
new FriedRice().cook();
new Noodle().cook();
```

### 应用场景

- Spring 的 `JdbcTemplate`、`RedisTemplate`
- HttpServlet 的 `doGet`、`doPost`

# 设计模式总结

## 怎么选用？

| 你的需求             | 推荐模式      |
| ---------------- | --------- |
| 全局只要一个对象         | **单例**    |
| 创建对象的逻辑复杂        | **工厂**    |
| 对象参数太多，构造方法乱     | **建造者**   |
| 想给对象加功能，又不想动原代码  | **装饰器**   |
| 想增强方法（日志、事务）     | **代理**    |
| 接口不兼容            | **适配器**   |
| 一堆 if-else 选算法   | **策略**    |
| 状态变化要通知多个对象      | **观察者**   |
| 流程固定，部分步骤可变      | **模板方法**  |

## 学习建议

1. **不要为了用而用** —— 设计模式是为了解决问题，不是炫技
2. **先学三五个最常用的**（单例、工厂、策略、代理、观察者）
3. **结合源码看** —— Spring、JDK 源码到处都是设计模式的应用
4. **多写多用才能内化** —— 看懂 ≠ 会用

## 常被一起问的"模式区别"

| 对比                | 区别要点                            |
| ----------------- | ------------------------------- |
| **代理 vs 装饰器**     | 代理控制访问，装饰器增强功能                  |
| **策略 vs 状态**      | 策略由外部切换，状态由对象内部驱动               |
| **工厂 vs 建造者**     | 工厂关注"造什么"，建造者关注"怎么一步步造"          |
| **简单工厂 vs 工厂方法**  | 简单工厂一个类管所有，工厂方法每种产品一个工厂         |
