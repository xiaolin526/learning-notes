# Java 多线程基础

让程序"同时"做多件事的能力。多核 CPU 时代必备技能，也是面试高频考点。

## 一、进程与线程

| 概念  | 说明                                           |
| --- | -------------------------------------------- |
| 进程  | 操作系统分配资源的基本单位（一个 QQ、一个浏览器就是一个进程）             |
| 线程  | 进程内的执行单元，**共享进程的内存**（一个 QQ 里聊天、传文件可以是不同线程）  |

**为什么需要多线程？**
- 充分利用多核 CPU
- 提高响应速度（UI 不卡顿）
- 处理并发任务（一个服务器同时响应多个用户）

## 二、创建线程的三种方式

### 方式 1：继承 Thread

```java
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程运行：" + Thread.currentThread().getName());
    }
}

// 使用
MyThread t = new MyThread();
t.start();   // ⚠️ 调用 start() 而不是 run()！
```

### 方式 2：实现 Runnable 接口（推荐）

```java
public class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("任务执行中");
    }
}

// 使用
Thread t = new Thread(new MyTask());
t.start();

// 用 Lambda 更简洁
new Thread(() -> System.out.println("Hello from thread")).start();
```

### 方式 3：实现 Callable 接口（有返回值）

```java
import java.util.concurrent.*;

Callable<Integer> task = () -> {
    int sum = 0;
    for (int i = 1; i <= 100; i++) sum += i;
    return sum;
};

FutureTask<Integer> future = new FutureTask<>(task);
new Thread(future).start();

Integer result = future.get();   // 阻塞等待结果
System.out.println(result);       // 5050
```

### 三种方式对比

| 方式       | 优点                  | 缺点          |
| -------- | ------------------- | ----------- |
| Thread   | 简单                  | 单继承限制       |
| Runnable | 灵活，可与线程池配合          | 没有返回值       |
| Callable | 有返回值，可抛异常           | 略复杂         |

> 💡 **实际开发中优先用 Runnable / Callable + 线程池**，几乎不直接 new Thread。

## 三、start() vs run()

```java
Thread t = new Thread(() -> System.out.println(Thread.currentThread().getName()));

t.run();     // 直接调用，在主线程执行（输出 "main"）
t.start();   // 启动新线程执行（输出 "Thread-0"）
```

**记住：要新开线程必须用 `start()`，调用 `run()` 只是普通方法调用。**

## 四、线程的生命周期

```
NEW（新建）
  ↓ start()
RUNNABLE（可运行）  ←→  BLOCKED（阻塞，等待锁）
  ↓                  WAITING（无限期等待，wait/join）
  ↓                  TIMED_WAITING（限时等待，sleep）
TERMINATED（终止）
```

## 五、常用方法

### sleep —— 让线程睡一会儿

```java
try {
    Thread.sleep(1000);   // 睡 1 秒
} catch (InterruptedException e) {
    e.printStackTrace();
}
```

### join —— 等待另一个线程结束

```java
Thread t = new Thread(() -> {
    try { Thread.sleep(2000); } catch (InterruptedException e) {}
    System.out.println("子线程结束");
});
t.start();
t.join();   // 主线程在这里等 t 执行完
System.out.println("主线程继续");
```

### yield —— 让出 CPU（建议性，不一定生效）

```java
Thread.yield();   // 提示调度器：我可以让出 CPU
```

### interrupt —— 中断线程

```java
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 干活
    }
    System.out.println("被中断了，退出");
});
t.start();
Thread.sleep(1000);
t.interrupt();   // 设置中断标志
```

## 六、线程安全问题

**核心问题：** 多个线程同时修改共享数据时会出错。

```java
public class Counter {
    private int count = 0;

    public void increment() {
        count++;   // ⚠️ 这一行不是原子操作！
    }
}

// 1000 个线程各自调用 1000 次 increment()
// 期望结果：1000000
// 实际结果：往往小于 1000000
```

**为什么？** `count++` 实际上是三步：
1. 读取 count 的值
2. 加 1
3. 写回 count

多个线程交叉执行这三步就会丢失更新。

## 七、synchronized —— 最基础的同步

### 同步方法

```java
public class Counter {
    private int count = 0;

    // 加上 synchronized，整个方法变成原子操作
    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}
```

### 同步代码块

```java
public class Counter {
    private int count = 0;
    private final Object lock = new Object();

    public void increment() {
        synchronized (lock) {
            count++;
        }
    }
}
```

### synchronized 的本质

- 锁的是**对象**（实例方法锁的是 this，静态方法锁的是类的 Class 对象）
- 同一时刻只允许一个线程持有锁
- 是**可重入锁**（同一线程可重复获得同一把锁）

```java
public synchronized void a() {
    b();   // OK，同一线程可以再次进入 b()
}
public synchronized void b() {
    // ...
}
```

## 八、volatile 关键字

保证变量的**可见性**，但**不保证原子性**。

```java
public class Flag {
    private volatile boolean running = true;

    public void stop() {
        running = false;   // 其他线程能立即看到这个修改
    }

    public void run() {
        while (running) {
            // 干活
        }
    }
}
```

> ⚠️ `volatile` 不能替代 `synchronized`，它**不能保证 `count++` 这种复合操作的原子性**。

## 九、原子类（推荐！）

`java.util.concurrent.atomic` 包提供了一系列原子操作类。

```java
import java.util.concurrent.atomic.AtomicInteger;

public class Counter {
    private AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();   // 原子操作，线程安全
    }

    public int get() {
        return count.get();
    }
}
```

常用原子类：
- `AtomicInteger` / `AtomicLong` / `AtomicBoolean`
- `AtomicReference<T>`
- `LongAdder`（高并发计数性能更好）

## 十、Lock 接口（更灵活的锁）

```java
import java.util.concurrent.locks.*;

public class Counter {
    private final Lock lock = new ReentrantLock();
    private int count = 0;

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();   // 必须放在 finally！
        }
    }
}
```

### ReentrantLock vs synchronized

| 对比项     | synchronized | ReentrantLock     |
| ------- | ------------ | ----------------- |
| 关键字/类   | 关键字          | 类                 |
| 自动释放    | ✅ 自动         | ❌ 必须手动 unlock     |
| 可中断     | ❌            | ✅ `lockInterruptibly()` |
| 公平锁     | ❌（非公平）       | ✅ 支持              |
| 尝试获取    | ❌            | ✅ `tryLock()`     |
| 性能      | JDK 优化后差不多   | 差不多               |

## 十一、线程通信：wait / notify

```java
public class Box {
    private int value;
    private boolean hasValue = false;

    public synchronized void put(int v) throws InterruptedException {
        while (hasValue) {
            wait();   // 释放锁，等待
        }
        value = v;
        hasValue = true;
        notifyAll();   // 唤醒所有等待的线程
    }

    public synchronized int take() throws InterruptedException {
        while (!hasValue) {
            wait();
        }
        hasValue = false;
        notifyAll();
        return value;
    }
}
```

> `wait` / `notify` 必须在 `synchronized` 块内调用！

## 十二、线程池（重点！）

**为什么用线程池？**
- 避免频繁创建/销毁线程的开销
- 控制并发数量
- 统一管理任务

### 创建线程池

```java
import java.util.concurrent.*;

ExecutorService pool = Executors.newFixedThreadPool(4);

// 提交任务
pool.submit(() -> System.out.println("任务 1"));
pool.submit(() -> System.out.println("任务 2"));

// 关闭线程池
pool.shutdown();   // 等已提交任务执行完
// pool.shutdownNow();   // 立即停止
```

### 常用线程池

| 工厂方法                          | 说明                |
| ----------------------------- | ----------------- |
| `newFixedThreadPool(n)`       | 固定大小              |
| `newCachedThreadPool()`       | 按需创建，空闲回收（小心 OOM） |
| `newSingleThreadExecutor()`   | 单线程顺序执行           |
| `newScheduledThreadPool(n)`   | 支持定时/周期任务         |

### 阿里巴巴规范：用 ThreadPoolExecutor 自己创建

```java
ThreadPoolExecutor pool = new ThreadPoolExecutor(
    2,                          // 核心线程数
    4,                          // 最大线程数
    60L, TimeUnit.SECONDS,      // 空闲存活时间
    new LinkedBlockingQueue<>(100),   // 任务队列
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.AbortPolicy()   // 拒绝策略
);
```

> 因为 `Executors` 提供的快捷方法**默认队列无界**，可能导致 OOM。

### 提交任务的两种方式

```java
// execute：无返回值
pool.execute(() -> System.out.println("跑"));

// submit：返回 Future，可获取结果
Future<Integer> future = pool.submit(() -> 1 + 2);
Integer result = future.get();
```

## 十三、并发容器

线程安全的集合：

```java
// 替代 HashMap
Map<String, Integer> map = new ConcurrentHashMap<>();

// 替代 ArrayList
List<String> list = new CopyOnWriteArrayList<>();

// 阻塞队列（生产者-消费者）
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
queue.put(task);     // 满了会阻塞
Task t = queue.take();   // 空了会阻塞
```

## 十四、CountDownLatch —— 倒计时门闩

让一个线程等待其他若干线程完成后再继续。

```java
CountDownLatch latch = new CountDownLatch(3);

for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        System.out.println("子线程干活");
        latch.countDown();   // 计数 -1
    }).start();
}

latch.await();   // 等待计数归零
System.out.println("所有子线程完成，主线程继续");
```

## 十五、最佳实践

✅ **该做的：**
- 优先使用线程池，不要 `new Thread()`
- 共享变量用 `AtomicXxx` 或并发容器
- `synchronized` 块越小越好（减少锁竞争）
- `lock.unlock()` 一定放在 `finally`
- 多线程下用 `ConcurrentHashMap` 而非 `HashMap`

❌ **不该做的：**
- 不要用 `Thread.stop()`（已废弃，会导致数据不一致）
- 不要在循环里频繁创建线程
- 不要忽略 `InterruptedException`
- 不要在持有锁时调用未知代码（容易死锁）

## 十六、死锁示例（反面教材）

```java
Object lockA = new Object();
Object lockB = new Object();

// 线程 1
new Thread(() -> {
    synchronized (lockA) {
        try { Thread.sleep(100); } catch (Exception e) {}
        synchronized (lockB) {
            System.out.println("线程 1 拿到了两把锁");
        }
    }
}).start();

// 线程 2
new Thread(() -> {
    synchronized (lockB) {
        try { Thread.sleep(100); } catch (Exception e) {}
        synchronized (lockA) {
            System.out.println("线程 2 拿到了两把锁");
        }
    }
}).start();

// 结果：两个线程互相等对方释放锁，永远卡住
```

**避免死锁：** 所有线程**按相同顺序**获取锁。
