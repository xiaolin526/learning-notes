# 艾宾浩斯遗忘曲线复习调度 · 讲解手册

> 主线:**「一道题的一生」**。暗线:**三个字段**(reviewStage / reviewDate / isCompleted)。
> 全程约 4–5 分钟,重头戏是 **第 2 步(间隔数组)** 和 **第 5 步(状态机)**,这两页多停留,其余快速带过。

---

## 第 1 步 · 抛出问题:遗忘曲线 *(约 40 秒)*

**核心一句:** 学完就忘,但在快忘时复习一下,就能把记忆拉回来,而且越来越耐忘。

**怎么讲:** 先指红色虚线「什么都不做,记忆掉得很快」,再指绿色实线「在对的时间点复习,每次拉回顶部,下坡越来越缓」。先把「为什么需要这套算法」立住。

**配图:** 红绿曲线图。本页无代码。

---

## 第 2 步 · 把直觉变成数字:间隔数组 *(约 30 秒)*

**核心一句:** 复习不是天天复习,而是按 1、2、4、7、15、30 天这个越来越稀的节奏。

**怎么讲:** 亮出间隔数组,说「这 6 个数就是曲线上那几个绿点,代码把它延伸到了 30 天。间隔越往后越长——因为每复习一次就更耐忘,可以隔更久再来」。这是整个算法的灵魂,值得单独一页。

**高亮:** 整段(尤其那一行数组)。

```java
/** 遗忘曲线的复习间隔天数(索引 0 不用,1-6 对应 6 个阶段) */
private static final int[] REVIEW_INTERVALS = {0, 1, 2, 4, 7, 15, 30};
/** 最大复习阶段 */
private static final int MAX_STAGE = 6;
```

---

## 第 3 步 · 数据怎么存:三个字段 *(约 30 秒)*

**核心一句:** 整个算法,就是在反复更新一道题的三个字段。

**怎么讲:** `reviewStage`(我现在在曲线的第几个点)、`reviewDate`(下次哪天该复习)、`isCompleted`(这次做完没)。先把这三个字段立成「主角」,后面所有逻辑都只是在改它们,听众就有了挂钩子的地方。

**高亮:** `reviewStage` / `reviewDate` / `isCompleted` 三行。

```java
@Schema(description = "复习阶段:1=1天后 2=2天后 3=4天后 4=7天后 5=15天后 6=30天后")
private Integer reviewStage;

@Schema(description = "计划复习日期")
private LocalDate reviewDate;

@Schema(description = "是否已完成:0=待复习 1=已完成")
private Integer isCompleted;
```

---

## 第 4 步 · 入口:答题怎么触发 *(约 30 秒)*

**核心一句:** 只有「没记住」的题,才会被丢进遗忘曲线机制。

**怎么讲:** 对着流程图上半部分讲 `processAnswer`——答对了不进循环(顶多 `markIfMastered` 检查一下是否已掌握);答错才 `addToWrongBook` 进错题本,并 `createReviewPlan(...,1)` 从阶段 1 开始排计划。

**高亮:** `if (!isCorrect)` 这条分支。

**配图:** 流程图最上面那段(到「答错→创建阶段1」为止)。

```java
@Override
@Transactional
public void processAnswer(UserAnswer answer) {
    Integer userId = answer.getUserId();
    Integer questionId = answer.getQuestionId();
    boolean isCorrect = answer.getIsCorrect() == 1;
    if (!isCorrect) {
        // 答错 → 加入错题本 + 从阶段1开始建复习计划
        addToWrongBook(userId, questionId);
        createReviewPlan(userId, questionId, 1);
    } else {
        // 答对 → 检查是否可标记掌握
        markIfMastered(userId, questionId);
    }
}
```

## 第 5 步 · 核心:completeReview 状态机 *(约 90 秒,重头戏)*

**核心一句:** 每复习一次,只有三种结局:晋级、毕业、或打回重来。

**怎么讲:** 先说开头的幂等保护(`plan==null || isCompleted==1` 就 return,防重复提交),再说「先把本次标记为已完成」,然后三条岔路:

- 复习答对 → 晋级到下一阶段,按更长间隔排下次(对应曲线「拉回顶部、坡变缓」)
- 答对且已是第 6 阶段(`nextStage > MAX_STAGE`) → `markAsMastered` 毕业,退出循环
- 复习答错 → 阶段打回 1,明天重来(对应曲线「没记住又掉下去」)

**讲法建议:** 带着听众走一遍「一道题的一生」——答错入列 → 阶段1 → 一路答对晋级到阶段6 → 毕业;再回头点那条红线(答错重置)和左上角的回环箭头(等到下次复习日再进 `completeReview`)。

**高亮:** `if (isCorrect)` 里的「晋级 / 毕业 / 重置」三条路。上半段(标记完成)可淡化,让注意力落在状态流转上。

**配图:** 流程图虚线框那一整块,或状态机图。

```java
@Override
@Transactional
public void completeReview(Integer planId, boolean isCorrect) {
    ReviewPlan plan = reviewPlanMapper.selectById(planId);
    if (plan == null || plan.getIsCompleted() == 1) {
        return;   // 计划不存在或已完成,直接返回(幂等保护)
    }
    // 把当前这次复习标记为已完成
    plan.setIsCompleted(1);
    plan.setCompletedTime(LocalDateTime.now());
    plan.setIsCorrect(isCorrect ? 1 : 0);
    reviewPlanMapper.updateById(plan);

    if (isCorrect) {
        int nextStage = plan.getReviewStage() + 1;
        if (nextStage > MAX_STAGE) {
            // 6 个阶段全部通过 → 标记"已掌握",退出复习循环
            markAsMastered(plan.getUserId(), plan.getQuestionId());
        } else {
            // 晋级 → 按更长的间隔安排下次复习
            createReviewPlan(plan.getUserId(), plan.getQuestionId(), nextStage);
        }
    } else {
        // 复习答错 → 阶段重置为 1,重新巩固
        addToWrongBook(plan.getUserId(), plan.getQuestionId());
        createReviewPlan(plan.getUserId(), plan.getQuestionId(), 1);
    }
}
```

**可选** — 配上 Controller 入口,顺带展示给学生的反馈文案(讲「用户体验」时很自然):

```java
@PostMapping("/complete/{planId}")
public Result<String> completeReview(@PathVariable Integer planId,
                                     @RequestParam boolean isCorrect) {
    reviewService.completeReview(planId, isCorrect);
    String message = isCorrect ? "回答正确!已安排下一阶段复习"
                               : "回答错误,复习阶段已重置,明天再来巩固";
    return Result.success(message, null);
}
```

## 第 6 步 · 算日期:createReviewPlan *(约 30 秒)*

**核心一句:** 下次复习日 = 今天 + 当前阶段对应的间隔。

**怎么讲:** 灵魂就一行 `reviewDate = LocalDate.now().plusDays(REVIEW_INTERVALS[stage])`。再补一句它会先把这道题「未完成的旧计划」作废,保证一道题同时只有一条有效计划,不会排重。

**高亮:** 「作废旧计划」和 `plusDays(REVIEW_INTERVALS[stage])` 两处。

```java
private void createReviewPlan(Integer userId, Integer questionId, int stage) {
    // 先把该用户该题目所有未完成的旧计划作废,避免重复
    reviewPlanMapper.update(null,
            new LambdaUpdateWrapper<ReviewPlan>()
                    .eq(ReviewPlan::getUserId, userId)
                    .eq(ReviewPlan::getQuestionId, questionId)
                    .eq(ReviewPlan::getIsCompleted, 0)
                    .set(ReviewPlan::getIsCompleted, 1));

    // 关键:复习日期 = 今天 + 该阶段对应的间隔天数
    LocalDate reviewDate = LocalDate.now().plusDays(REVIEW_INTERVALS[stage]);

    Question question = questionMapper.selectById(questionId);
    ReviewPlan plan = new ReviewPlan();
    plan.setUserId(userId);
    plan.setQuestionId(questionId);
    plan.setKnowledgeId(question != null ? question.getKnowledgeId() : 0);
    plan.setReviewDate(reviewDate);
    plan.setReviewStage(stage);
    plan.setIsCompleted(0);
    reviewPlanMapper.insert(plan);
}
```

## 第 7 步 · 怎么落地:getTodayReviewPlans *(约 30 秒)*

**核心一句:** 每天把「到期该复习」的题捞出来,推给学生。

**怎么讲:** `le(reviewDate, today)` 捞出「今天及之前、还没复习的」,驱动今日任务和复习日历。强调是「及之前」——所以哪天忘了复习也不会丢,会累积到今天等你处理。这也顺带回答了「系统怎么知道今天该复习哪些题」。

**高亮:** `le(ReviewPlan::getReviewDate, LocalDate.now())` 这行。

```java
@Override
public List<ReviewPlan> getTodayReviewPlans(Integer userId) {
    List<ReviewPlan> plans = reviewPlanMapper.selectList(
            new LambdaQueryWrapper<ReviewPlan>()
                    .eq(ReviewPlan::getUserId, userId)
                    .eq(ReviewPlan::getIsCompleted, 0)
                    .le(ReviewPlan::getReviewDate, LocalDate.now())  // 今天及之前未完成
                    .orderByAsc(ReviewPlan::getReviewDate));
    for (ReviewPlan plan : plans) {
        plan.setQuestion(questionMapper.selectById(plan.getQuestionId()));
    }
    return plans;
}
```

## 第 8 步 · 收尾:闭环 *(约 20 秒)*

**核心一句:** 曲线图说「为什么」,流程图说「怎么做」。

**怎么讲:** 把两张图连起来收口——绿点的位置 = `REVIEW_INTERVALS`;曲线被拉回顶部 = 答对晋级;掉下去重来 = 答错重置。听众的认知就闭环了。

## 排版小贴士

- 每页代码别超过一屏,**一屏一段 + 一句核心话**最好讲。
- `completeReview` 偏长,把上半段(标记完成)淡化,只高亮下半段三条分支。
- 抓住「三个字段 + 一道题的一生」这条线,听众就不会被细节淹没。
