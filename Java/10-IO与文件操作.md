# Java IO 与文件操作

**IO（Input/Output）** 是程序和外部世界（文件、网络、控制台）交换数据的桥梁。Java 的 IO 体系非常庞大，但理解了核心套路就不难。

## 一、IO 流的分类

### 按方向

| 类型     | 说明           |
| ------ | ------------ |
| 输入流（Input） | 从外部 → 程序     |
| 输出流（Output）| 从程序 → 外部     |

### 按数据单位

| 类型     | 单位       | 适用                |
| ------ | -------- | ----------------- |
| 字节流    | 8 位字节    | 图片、音频、视频等**二进制文件** |
| 字符流    | 16 位字符   | **文本文件**          |

### 四大基类

```
        InputStream    OutputStream    （字节流，抽象类）
            ↑                ↑
       ┌────┴────┐     ┌─────┴─────┐
   FileInputStream  FileOutputStream  ...

         Reader          Writer       （字符流，抽象类）
            ↑                ↑
       ┌────┴────┐     ┌─────┴─────┐
     FileReader      FileWriter     ...
```

> 💡 **记忆口诀：** 二进制用字节流，文本用字符流。

## 二、File 类 —— 文件和目录的抽象

`File` 表示文件或目录的路径，**不涉及内容读写**。

```java
import java.io.File;

File f = new File("test.txt");

// 创建文件
if (!f.exists()) {
    f.createNewFile();
}

// 信息查询
f.exists();        // 是否存在
f.isFile();        // 是文件吗
f.isDirectory();   // 是目录吗
f.length();        // 大小（字节）
f.getName();       // 文件名
f.getAbsolutePath();  // 绝对路径
f.lastModified();  // 最后修改时间

// 创建目录
File dir = new File("data/logs");
dir.mkdirs();      // 创建多级目录（mkdir 只能建一级）

// 列出子文件
File folder = new File(".");
String[] files = folder.list();
File[] fileObjs = folder.listFiles();

// 删除
f.delete();        // 删除文件或空目录
```

## 三、字节流：读写二进制文件

### FileInputStream / FileOutputStream

```java
import java.io.*;

// 读取文件
try (FileInputStream fis = new FileInputStream("input.bin")) {
    int b;
    while ((b = fis.read()) != -1) {   // 每次读 1 个字节，-1 表示结束
        System.out.print(b + " ");
    }
}

// 写入文件
try (FileOutputStream fos = new FileOutputStream("output.bin")) {
    fos.write(65);              // 写一个字节（A）
    fos.write(new byte[]{66, 67, 68});  // 写一个字节数组（BCD）
}
```

### 缓冲流（推荐！性能高 N 倍）

```java
// 一次读一个字节太慢，用缓冲流批量读
try (BufferedInputStream bis = new BufferedInputStream(
         new FileInputStream("input.bin"));
     BufferedOutputStream bos = new BufferedOutputStream(
         new FileOutputStream("output.bin"))) {

    byte[] buffer = new byte[1024];
    int len;
    while ((len = bis.read(buffer)) != -1) {
        bos.write(buffer, 0, len);   // 注意 0, len，不要写整个 buffer
    }
}
```

> ⚡ **缓冲流的速度通常是普通流的 10 倍以上。**

## 四、字符流：读写文本文件

### FileReader / FileWriter

```java
// 读取
try (FileReader fr = new FileReader("test.txt")) {
    int c;
    while ((c = fr.read()) != -1) {
        System.out.print((char) c);
    }
}

// 写入
try (FileWriter fw = new FileWriter("test.txt")) {
    fw.write("Hello, 世界！");
    fw.write("\n第二行");
}
```

### BufferedReader / BufferedWriter（强烈推荐）

```java
// 按行读取
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

// 按行写入
try (BufferedWriter bw = new BufferedWriter(new FileWriter("test.txt"))) {
    bw.write("第一行");
    bw.newLine();              // 写入换行符（跨平台）
    bw.write("第二行");
}
```

## 五、字符编码问题（超重要！）

文本文件读写最容易出问题的就是**编码**——不指定编码，可能乱码。

```java
// ❌ 用平台默认编码（Windows 是 GBK，Linux 是 UTF-8）
new FileReader("test.txt");

// ✅ 显式指定 UTF-8
new InputStreamReader(new FileInputStream("test.txt"), "UTF-8");

// ✅ 完整链路
try (BufferedReader br = new BufferedReader(
        new InputStreamReader(
            new FileInputStream("test.txt"),
            StandardCharsets.UTF_8))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}
```

> 💡 **现代 Java（11+）有更简单的写法**，看下面 NIO 部分。

## 六、转换流：字节流 → 字符流

```java
// InputStreamReader：把字节流"翻译"成字符流
// OutputStreamWriter：反过来

InputStreamReader isr = new InputStreamReader(
    System.in,                  // 字节流（标准输入）
    StandardCharsets.UTF_8      // 指定编码
);
BufferedReader br = new BufferedReader(isr);

System.out.print("请输入：");
String line = br.readLine();
System.out.println("你输入的是：" + line);
```

## 七、对象序列化

把对象保存到文件 / 在网络上传输，对象需要实现 `Serializable` 接口。

```java
public class Person implements Serializable {
    private static final long serialVersionUID = 1L;   // 推荐显式声明
    private String name;
    private int age;
    private transient String password;   // transient 修饰的字段不会被序列化
    // 构造方法、getter/setter ...
}

// 序列化（写入）
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("person.dat"))) {
    Person p = new Person("小林", 18);
    oos.writeObject(p);
}

// 反序列化（读取）
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("person.dat"))) {
    Person p = (Person) ois.readObject();
    System.out.println(p.getName());
}
```

> ⚠️ 注意事项：
> - 类必须实现 `Serializable`
> - `transient` 字段不会序列化
> - **`serialVersionUID` 不一致会导致反序列化失败**
> - 现代项目更多用 JSON（Jackson、Gson）替代 Java 原生序列化

## 八、NIO（New IO，Java 1.4+）

NIO 比传统 IO 更高效，提供了 **Buffer + Channel + Selector** 的新模型。但日常用得最多的是它的**便捷工具类**。

### Files 工具类（Java 7+，超好用！）

```java
import java.nio.file.*;
import java.nio.charset.StandardCharsets;

Path path = Paths.get("test.txt");

// ① 一行代码读全部内容
String content = Files.readString(path, StandardCharsets.UTF_8);   // Java 11+

// ② 一行代码读所有行
List<String> lines = Files.readAllLines(path, StandardCharsets.UTF_8);

// ③ 一行代码写入
Files.writeString(path, "Hello", StandardCharsets.UTF_8);          // Java 11+
Files.write(path, List.of("行1", "行2"), StandardCharsets.UTF_8);

// ④ 流式读取大文件（不会 OOM）
try (Stream<String> stream = Files.lines(path, StandardCharsets.UTF_8)) {
    stream.filter(line -> line.contains("error"))
          .forEach(System.out::println);
}

// ⑤ 文件操作
Files.exists(path);
Files.size(path);
Files.copy(src, dst);
Files.move(src, dst);
Files.delete(path);

// ⑥ 创建目录
Files.createDirectories(Paths.get("data/logs"));

// ⑦ 列出目录
try (Stream<Path> stream = Files.list(Paths.get("."))) {
    stream.forEach(System.out::println);
}

// ⑧ 递归遍历
try (Stream<Path> stream = Files.walk(Paths.get("."))) {
    stream.filter(p -> p.toString().endsWith(".java"))
          .forEach(System.out::println);
}
```

> 💡 **现代 Java 开发中，文件操作首选 `Files` 工具类**，比传统 IO 简洁 N 倍。

## 九、实战示例

### 示例 1：复制文件

```java
public static void copyFile(String src, String dst) throws IOException {
    // 现代写法（Java 7+）
    Files.copy(Paths.get(src), Paths.get(dst), StandardCopyOption.REPLACE_EXISTING);
}
```

### 示例 2：统计文件行数

```java
public static long countLines(String path) throws IOException {
    try (Stream<String> stream = Files.lines(Paths.get(path), StandardCharsets.UTF_8)) {
        return stream.count();
    }
}
```

### 示例 3：日志文件按行处理

```java
public static void grepErrors(String logPath) throws IOException {
    try (Stream<String> stream = Files.lines(Paths.get(logPath), StandardCharsets.UTF_8)) {
        stream.filter(line -> line.contains("ERROR"))
              .map(line -> "[告警] " + line)
              .forEach(System.out::println);
    }
}
```

### 示例 4：递归删除目录

```java
public static void deleteDirectory(Path dir) throws IOException {
    try (Stream<Path> walk = Files.walk(dir)) {
        walk.sorted(Comparator.reverseOrder())   // 先删子文件，再删父目录
            .forEach(p -> {
                try {
                    Files.delete(p);
                } catch (IOException e) {
                    e.printStackTrace();
                }
            });
    }
}
```

## 十、关于 try-with-resources

**所有 IO 操作都必须正确关闭流**，否则会**资源泄漏**。

```java
// ❌ 旧写法，繁琐还容易忘记关闭
FileInputStream fis = null;
try {
    fis = new FileInputStream("test.txt");
    // ...
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (fis != null) {
        try { fis.close(); } catch (IOException e) {}
    }
}

// ✅ 现代写法，自动关闭
try (FileInputStream fis = new FileInputStream("test.txt")) {
    // ...
} catch (IOException e) {
    e.printStackTrace();
}
```

## 十一、Properties 配置文件

Java 项目里常见的 `.properties` 文件读取方式：

```properties
# config.properties
db.url=jdbc:mysql://localhost:3306/test
db.user=root
db.password=123456
```

```java
Properties props = new Properties();
try (InputStream in = new FileInputStream("config.properties")) {
    props.load(in);
}

String url = props.getProperty("db.url");
String user = props.getProperty("db.user");
String password = props.getProperty("db.password");
```

## 十二、最佳实践

✅ **该做的：**
- **永远使用 try-with-resources** 保证流被关闭
- **永远使用缓冲流**（`BufferedReader/Writer`）提升性能
- **永远显式指定字符编码**（推荐 UTF-8）
- 现代代码优先用 `Files` / `Path` 而非 `File` / `FileInputStream`
- 大文件用 `Files.lines()` 流式处理，不要一次读全部

❌ **不该做的：**
- 不要用 `FileReader`/`FileWriter`（不能指定编码，容易乱码）
- 不要忘记关闭流
- 不要一次性 `readAllBytes` 大文件（OOM 警告）
- 不要在循环里频繁打开/关闭流

## 十三、IO vs NIO vs AIO

| 模型  | 特点                  | 适用                  |
| --- | ------------------- | ------------------- |
| BIO | 同步阻塞，一连接一线程         | 连接数少的传统场景           |
| NIO | 同步非阻塞，一线程管理多连接（Selector）| **高并发**，如 Netty     |
| AIO | 异步非阻塞，事件回调          | Linux 支持有限，实际用得少    |

> 💡 高并发服务器（聊天室、游戏服务端、Web 服务器）几乎都基于 **NIO**。**Netty** 是 Java NIO 的事实标准框架。
