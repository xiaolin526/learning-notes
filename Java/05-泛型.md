
# Java 泛型

**泛型（Generics）** 让类、接口、方法可以**操作任意类型**，同时在编译期就检查类型安全，避免运行时出错。

## 一、为什么需要泛型？

### 没有泛型的痛苦

```java
// 一个能装"任何东西"的列表
List list = new ArrayList();
list.add("Hello");
list.add(123);          // 不小心放进了一个 Integer
list.add(true);         // 又混进一个 Boolean

// 取出来时必须强转
String s = (String) list.get(1);   // ❌ 运行时崩溃 ClassCastException
```

**问题：**
- 编译时不报错
- 运行时才暴露问题
- 取数据要手动强转

### 有泛型之后

```java
List<String> list = new ArrayList<>();
list.add("Hello");
list.add(123);          // ❌ 编译时就报错，根本进不去

String s = list.get(0); // 自动是 String，不用强转
```

**好处：**
- ✅ 编译期类型检查
- ✅ 自动类型转换
- ✅ 代码更清晰可读

## 二、泛型类

定义类时用 `<T>` 占位符表示"某种类型"：

```java
// T 是类型参数（可以叫任何名字，习惯用 T、E、K、V）
public class Box<T> {
    private T value;

    public void set(T value) {
        this.value = value;
    }

    public T get() {
        return value;
    }
}

// 使用
Box<String> sBox = new Box<>();
sBox.set("Hello");
String s = sBox.get();          // 不用强转

Box<Integer> iBox = new Box<>();
iBox.set(123);
int n = iBox.get();
```

> 💡 创建对象时，右边的 `<>` 里类型可以省略（Java 7+ 的"菱形语法"）：
> `new Box<>()` 而不是 `new Box<String>()`

## 三、多个类型参数

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey()   { return key; }
    public V getValue() { return value; }
}

// 使用
Pair<String, Integer> p = new Pair<>("年龄", 18);
System.out.println(p.getKey() + " = " + p.getValue());
```

## 四、常用类型参数命名约定

| 字母 | 含义        | 常见用途              |
| -- | --------- | ----------------- |
| T  | Type      | 普通类型              |
| E  | Element   | 集合中的元素            |
| K  | Key       | Map 的键            |
| V  | Value     | Map 的值            |
| N  | Number    | 数字类型              |
| R  | Result    | 返回值类型             |

## 五、泛型方法

方法本身可以有自己的类型参数，**写在返回值前面**：

```java
public class Util {
    // <T> 声明这是个泛型方法
    public static <T> void print(T value) {
        System.out.println(value);
    }

    // 多个参数
    public static <T> T getFirst(List<T> list) {
        return list.get(0);
    }
}

// 使用
Util.print("Hello");        // 自动推断 T = String
Util.print(123);            // 自动推断 T = Integer

List<String> names = List.of("Alice", "Bob");
String first = Util.getFirst(names);   // first = "Alice"
```

## 六、泛型接口

```java
public interface Container<T> {
    void add(T item);
    T get(int index);
}

// 实现时指定具体类型
public class StringContainer implements Container<String> {
    private List<String> list = new ArrayList<>();

    @Override
    public void add(String item) {
        list.add(item);
    }

    @Override
    public String get(int index) {
        return list.get(index);
    }
}

// 或者继续保留泛型
public class MyContainer<T> implements Container<T> {
    // ...
}
```

## 七、通配符 `?`

当你**只是要使用**一个泛型对象，但不关心具体类型时，用 `?`。

### 上界通配符 `<? extends T>` —— "T 或 T 的子类"

```java
// 接受 List<Number>、List<Integer>、List<Double> 等
public static double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(1.1, 2.2);
sum(ints);      // ✅ OK
sum(doubles);   // ✅ OK
```

### 下界通配符 `<? super T>` —— "T 或 T 的父类"

```java
// 可以往里加 Integer 类型
public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}

List<Number> nums = new ArrayList<>();
addNumbers(nums);   // ✅ Number 是 Integer 的父类
```

### PECS 原则（记住这个！）

> **Producer Extends, Consumer Super**
> - 只读取数据用 `extends`（数据生产者）
> - 只写入数据用 `super`（数据消费者）

```java
// 从 src 读取（生产者）→ extends
// 往 dst 写入（消费者）→ super
public static <T> void copy(List<? extends T> src, List<? super T> dst) {
    for (T item : src) {
        dst.add(item);
    }
}
```

### 无界通配符 `<?>` —— "任意类型"

```java
// 只是遍历打印，不关心具体类型
public static void printList(List<?> list) {
    for (Object o : list) {
        System.out.println(o);
    }
}
```

## 八、类型擦除（重要！）

**Java 的泛型是"伪泛型"**，编译后类型信息会被擦除。

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();

System.out.println(list1.getClass() == list2.getClass());   // true
// 编译后两者都是 ArrayList，泛型信息消失了
```

### 类型擦除带来的限制

```java
// ❌ 不能用基本类型
List<int> list;          // 编译错误，要用 List<Integer>

// ❌ 不能创建泛型数组
T[] arr = new T[10];     // 编译错误

// ❌ 不能 instanceof 泛型
if (obj instanceof List<String>) { }   // 编译错误

// ❌ 静态成员不能用类的泛型参数
public class Box<T> {
    static T value;      // 编译错误
}
```

## 九、实战例子：通用的"返回结果"封装

后端开发里非常常见的写法：

```java
public class Result<T> {
    private int code;
    private String message;
    private T data;

    public Result(int code, String message, T data) {
        this.code = code;
        this.message = message;
        this.data = data;
    }

    // 静态工厂方法
    public static <T> Result<T> success(T data) {
        return new Result<>(200, "OK", data);
    }

    public static <T> Result<T> fail(String message) {
        return new Result<>(500, message, null);
    }

    // getter / setter ...
}

// 使用
Result<String> r1 = Result.success("Hello");
Result<List<User>> r2 = Result.success(userList);
Result<Void> r3 = Result.fail("用户不存在");
```

## 十、常见易错点

### 1. 泛型不支持协变

```java
List<Integer> ints = new ArrayList<>();
List<Number> nums = ints;    // ❌ 编译错误
// 即使 Integer 是 Number 的子类，List<Integer> 不是 List<Number> 的子类
```

要解决就用通配符：
```java
List<? extends Number> nums = ints;   // ✅ OK
```

### 2. 不能 new 泛型对象

```java
public class Box<T> {
    public void create() {
        T t = new T();   // ❌ 类型擦除导致无法 new
    }
}
```

绕开方式：传入 Class 对象
```java
public <T> T create(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();
}
```

### 3. 原始类型（Raw Type）

```java
List list = new ArrayList();      // 没指定泛型，叫"原始类型"
List<String> sList = list;        // ⚠️ 警告：unchecked
```

> 不要用原始类型，会失去泛型保护。
