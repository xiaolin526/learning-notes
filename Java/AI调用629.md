# AI 功能调用 · 代码详解

> 涉及文件：
> - `controller/AiController.java`（接口层）
> - `service/AiService.java`（接口定义层）
> - `service/impl/AiServiceImpl.java`（实现层 · 对接 DeepSeek 大模型）

四个 AI 功能：**1. 从笔记提取知识点　2. 按知识点自动出题　3. 流式答疑(SSE)　4. 简答题评分。**

---

## 一、核心套路（四个功能的共同地基）

四个功能本质是同一套路：**设计 prompt → 调 DeepSeek 的 `/v1/chat/completions` 接口 → 解析返回的内容 → 入库或返回**。区别只在 prompt 不同、以及"同步拿完整结果"还是"流式边出边推"。

### 1）大模型本身是无状态的

AI 不知道你在调用哪个功能，也不知道项目长什么样。它每次只做一件事：**收到一段 `messages`，生成一段回复**。

"是哪个功能"是**你的代码**靠 URL 路由分流决定的，然后给 AI 喂不同的 `systemPrompt`（系统提示词）。AI 是被提示词"附身"去扮演角色——读到"你是阅卷老师"就打分，读到"你是出题专家"就出题。**改 systemPrompt 等于换 AI 的人设和输出规矩**，是操控 AI 行为的唯一手柄。

也正因为无状态，答疑功能才要**手动**把历史对话塞回 `messages`（见功能 3）。

### 2）消息的两个角色

每次请求都拼一个 `messages` 列表，每条消息有两个固定字段：

```java
Map.of("role", "system", "content", systemPrompt)   // 角色=系统，内容=规则/人设
Map.of("role", "user",   "content", userPrompt)      // 角色=用户，内容=具体任务和数据
```

- `role` / `content` 是大模型 API 规定的**固定键名**，不能改。
- `"system"`/`"user"` 是从三种角色（system/user/assistant）里选的；`systemPrompt`/`userPrompt` 是你填的内容。
- 分工：`system` 放规则（你是谁、按什么格式输出），`user` 放具体内容。规则进 system，模型更当回事去遵守；且 system 写死、user 每次换内容，便于复用。

### 3）请求参数

```java
body.put("model", model);            // 用哪个模型，如 deepseek-chat
body.put("messages", messages);      // AI 要读的消息列表
body.put("max_tokens", maxTokens);   // 回复最多多长（4096），硬截断
body.put("temperature", temperature);// 发散程度（0.7），越高越随机
body.put("stream", false);           // false=一次性返回；true=流式逐段推
```

- `max_tokens`：1 token ≈ 1 个英文单词或 1~2 个汉字。是刹车也控成本，但**硬截断**——不够会写一半被砍，导致 JSON 残缺解析失败。
- `temperature`：低=稳定可复现（适合标准答案），高=发散有变化（适合创作）。0.7 是折中值。
- `stream`：提取/出题/评分用 false，答疑用 true。

### 4）JSON 解析的"扒皮"套路

提取/出题/评分都要求 AI 返回 JSON，但模型常包一层 Markdown 代码块，所以解析前先去壳：

```java
String json = response.replaceAll("```json\\s*", "").replaceAll("```\\s*", "").trim();
JsonNode array = objectMapper.readTree(json);
```

> 这套较脆：只要 AI 在 JSON 前后多说一句话，`readTree` 就抛异常，`catch` 里只 log 然后返回空。见文末"可优化项"。

## 二、调用链路

```
前端请求
   │
   ▼
AiController              ← 接参数、调服务、包结果（流式接口直接返回 SseEmitter）
   │  调用
   ▼
AiService（接口）          ← 声明 4 个能力
   │  实现
   ▼
AiServiceImpl            ← 拼 prompt、调大模型、解析、入库
   │  访问
   ▼
DeepSeek API  +  各 Mapper（笔记/知识点/题目/对话记录）
```

对外接口（路由前缀 `/ai`）：

| 方法 | URL | 作用 | 返回 |
|------|-----|------|------|
| `POST` | `/ai/extract/{noteId}` | 从笔记提取知识点 | `Result<List<KnowledgePoint>>` |
| `POST` | `/ai/generate/{knowledgeId}` | 自动出题 | `Result<List<Question>>` |
| `GET`  | `/ai/chat/stream` | 流式答疑 | `SseEmitter`（持续推送） |
| `POST` | `/ai/evaluate` | 简答题评分 | `Result<Map<String, Object>>` |

## 三、Controller 层详解（AiController）

```java
@RestController
@RequestMapping("/ai")
@Tag(name = "AI 智能接口")
public class AiController {

    @Autowired
    private AiService aiService;
```

### 1）提取知识点

```java
@PostMapping("/extract/{noteId}")
public Result<List<KnowledgePoint>> extractKnowledge(@PathVariable Integer noteId) {
    try {
        List<KnowledgePoint> points = aiService.extractKnowledgeFromNote(noteId);
        if (points.isEmpty()) {
            return Result.error("AI 提取失败，请重试");     // 空多半是 AI 返回异常
        }
        return Result.success("成功提取 " + points.size() + " 个知识点", points);
    } catch (RuntimeException e) {
        return Result.error(e.getMessage());            // 捕业务异常，返回错误信息
    }
}
```

- `@PathVariable Integer noteId`：取 URL 路径里的笔记 id。
- 结果为空 → 提示重试（通常是 AI 返回格式不规范，解析后列表为空）。
- `try-catch` 捕获 Service 抛的业务异常（如"笔记不存在"），转成错误响应。

### 2）自动出题

```java
@PostMapping("/generate/{knowledgeId}")
public Result<List<Question>> generateQuestions(
        @PathVariable Integer knowledgeId,
        @RequestParam(defaultValue = "1,2,3") String types,      // 题型，逗号分隔
        @RequestParam(defaultValue = "2") int countPerType) {    // 每种几道
    try {
        List<Integer> typeList = Arrays.stream(types.split(","))  // "1,2,3" 拆成数组转流
                .map(String::trim)                                // 去空格
                .map(Integer::parseInt)                           // 字符串转整数
                .toList();                                        // 收集成 List<Integer>

        List<Question> questions = aiService.generateQuestionsForKnowledge(knowledgeId, typeList, countPerType);
        if (questions.isEmpty()) {
            return Result.error("AI 出题失败，请重试");
        }
        return Result.success("成功生成 " + questions.size() + " 道题目", questions);
    } catch (RuntimeException e) {
        return Result.error(e.getMessage());
    }
}
```

- 题型参数前端传字符串 `"1,2,3"`，这里用 `split(",")` 拆开、`map(Integer::parseInt)` 逐个转成数字、`toList()` 收集成 `List<Integer>` 再传给服务。
- `@RequestParam(defaultValue = "...")`：不传则用默认值（题型默认 `1,2,3`，每种默认 2 道）。

### 3）流式答疑（与众不同：返回 SseEmitter）

```java
@GetMapping("/chat/stream")
public SseEmitter chatStream(
        @RequestParam Integer userId,
        @RequestParam(required = false) String sessionId,   // 会话 id，可空
        @RequestParam String message,                       // 提问内容
        @RequestParam(required = false) Integer subjectId) {

    SseEmitter emitter = new SseEmitter(120_000L);          // SSE 推送对象，超时 120 秒

    String session = (sessionId == null || sessionId.isBlank())  // 会话 id 为空
            ? UUID.randomUUID().toString() : sessionId;          // 则新建一个 UUID

    aiService.handleChatStream(emitter, userId, session, message, subjectId);  // 交服务异步处理
    return emitter;                                         // 立即返回，内容随后陆续推送
}
```

- 这个接口**不返回 `Result`，而是 `SseEmitter`**——它是"服务器推送"的载体，能持续往前端发数据，实现打字机效果。
- `SseEmitter(120_000L)`：超时 120 秒（`120_000` 毫秒，下划线只是数字分隔符，方便读）。
- `sessionId` 为空就用 `UUID.randomUUID()` 生成一个唯一会话号，用来隔离不同对话。
- 关键：`return emitter` **立即返回**，但内容还没生成完——服务在后台异步往 `emitter` 里推，前端逐段收到。

### 4）简答题评分

```java
@PostMapping("/evaluate")
public Result<Map<String, Object>> evaluateAnswer(
        @RequestParam String questionContent,   // 题目
        @RequestParam String correctAnswer,     // 标准答案
        @RequestParam String studentAnswer) {   // 学生作答
    return Result.success(aiService.evaluateAnswer(questionContent, correctAnswer, studentAnswer));
}
```

三个参数（题目、标准答案、学生答案）原样交给服务，返回评分结果。

## 四、Service 接口层详解（AiService）

```java
public interface AiService {

    /** 从指定笔记提取知识点 */
    List<KnowledgePoint> extractKnowledgeFromNote(Integer noteId);

    /** 为指定知识点生成题目 */
    List<Question> generateQuestionsForKnowledge(Integer knowledgeId, List<Integer> types, int count);

    /** AI 答疑对话（SSE 流式输出） */
    void handleChatStream(SseEmitter emitter, Integer userId, String sessionId, String message, Integer subjectId);

    /** AI 评价简答题答案 */
    Map<String, Object> evaluateAnswer(String questionContent, String correctAnswer, String studentAnswer);
}
```

注意 `handleChatStream` 返回 `void`（无返回值）——因为它不"返回"结果，而是通过参数里的 `emitter` 把内容**推**给前端。

## 五、Impl 层详解（AiServiceImpl）

### 配置与共用组件

```java
@Value("${deepseek.api-key}")  private String apiKey;       // 从配置注入：密钥
@Value("${deepseek.base-url}") private String baseUrl;      // 接口域名
@Value("${deepseek.model}")    private String model;        // 模型名
@Value("${deepseek.max-tokens}") private int maxTokens;     // 回复上限
@Value("${deepseek.temperature}") private double temperature;// 温度

private final ObjectMapper objectMapper = new ObjectMapper();   // JSON 处理器，全类共用

private final OkHttpClient httpClient = new OkHttpClient.Builder()  // HTTP 客户端
        .connectTimeout(30, TimeUnit.SECONDS)   // 连接超时 30 秒
        .readTimeout(120, TimeUnit.SECONDS)     // 读取超时 120 秒（AI 生成慢，给 2 分钟）
        .writeTimeout(30, TimeUnit.SECONDS)
        .build();
```

- `@Value("${...}")`：从配置文件 `application.yml` 读取值注入。这些参数都不写死在代码里。
- `objectMapper`：Jackson 的 JSON 工具，负责"对象 ↔ JSON 字符串"互转。
- `httpClient`：OkHttp 客户端，相当于代码里的"浏览器"。`readTimeout` 给到 120 秒，是因为大模型吐字慢，太短会中途断掉。

还注入了专门的线程池（供流式答疑异步使用）：

```java
@Autowired
@Qualifier("aiExecutor")          // 精确指定用名为 aiExecutor 的线程池
private Executor aiExecutor;      // AsyncConfig 中定义（核心5/最大20/队列50）
```

### 功能 1：从笔记提取知识点（信息抽取）

```java
public List<KnowledgePoint> extractKnowledgeFromNote(Integer noteId) {
    Note note = noteMapper.selectById(noteId);              // 查笔记
    if (note == null) throw new RuntimeException("笔记不存在");
    if (note.getContent() == null || note.getContent().isBlank())
        throw new RuntimeException("笔记内容为空，无法提取知识点");

    // 系统提示词：扮演教学助手，严格按 JSON 数组返回
    String systemPrompt = """
            你是一个教学助手，擅长从学习资料中提取核心知识点。
            …
            要求：
            1. 每个知识点包含一个简短的标题和一段摘要说明
            2. 知识点应该是独立的、可考核的概念
            3. 提取 3-8 个最重要的知识点
            4. 严格按以下 JSON 格式返回，不要有其他内容：
            [ {"title": "知识点标题", "summary": "知识点的详细说明，50-100字"} ]
            """;

    String response = callAi(systemPrompt, "请从以下笔记中提取知识点：\n\n" + note.getContent());

    List<KnowledgePoint> points = new ArrayList<>();
    try {
        String json = response.replaceAll("```json\\s*", "").replaceAll("```\\s*", "").trim();  // 扒皮
        JsonNode array = objectMapper.readTree(json);       // 解析成 JSON 数组

        for (JsonNode node : array) {                       // 遍历每个知识点
            KnowledgePoint kp = new KnowledgePoint();
            kp.setNoteId(noteId);
            kp.setSubjectId(note.getSubjectId());           // 学科继承自笔记，不让 AI 猜
            kp.setTitle(node.get("title").asText());        // 取 title 字段
            kp.setSummary(node.get("summary").asText());    // 取 summary 字段
            knowledgePointMapper.insert(kp);                // 边解析边入库
            points.add(kp);
        }
    } catch (Exception e) {
        log.error("解析知识点失败: {}", e.getMessage());      // 失败只 log，返回空列表
    }
    return points;
}
```

要点：

- 先校验笔记存在、内容非空，否则抛业务异常（被 Controller 的 try-catch 接住）。
- `systemPrompt` 用 Java 文本块（三引号 `"""`）写，设定 AI 角色 + 严格 JSON 格式。
- 用户消息 = 固定指令 + 笔记正文，体现"规则放 system、数据放 user"。
- 知识点的**学科直接继承自笔记**（`note.getSubjectId()`），不让 AI 猜。
- 解析后**逐条 `insert` 入库**，这一步是"出题"功能的前置（先有知识点才能出题）。

### 功能 2：为知识点自动出题（受控生成）

```java
public List<Question> generateQuestionsForKnowledge(Integer knowledgeId, List<Integer> types, int count) {
    KnowledgePoint kp = knowledgePointMapper.selectById(knowledgeId);
    if (kp == null) throw new RuntimeException("知识点不存在");

    String typeDesc = buildTypeDescription(types);          // 题型编码 [1,2,3] → 中文 "选择题、填空题、判断题"

    // 系统提示词：扮演出题专家，规则写得很死
    String systemPrompt = """
            你是一个出题专家，根据给定的知识点生成高质量的考试题目。
            …
            严格按以下 JSON 格式返回，不要有其他内容：
            [ { "type":1, "difficulty":2, "content":"题目内容",
                "options":["A.选项1","B.选项2","C.选项3","D.选项4"],
                "answer":"A", "explanation":"解析说明" } ]
            注意：
            - type: 1=选择 2=填空 3=判断 4=简答
            - difficulty: 1=简单 2=中等 3=困难
            - 填空题/判断题/简答题的 options 字段设为 null
            - 判断题的 answer 只能是 "正确" 或 "错误"
            """;

    String userPrompt = String.format(
            "知识点：%s\n知识点说明：%s\n\n请生成以下题型，每种 %d 道：%s",
            kp.getTitle(), kp.getSummary(), count, typeDesc);

    String response = callAi(systemPrompt, userPrompt);

    List<Question> questions = new ArrayList<>();
    try {
        String json = response.replaceAll("```json\\s*", "").replaceAll("```\\s*", "").trim();
        JsonNode array = objectMapper.readTree(json);

        for (JsonNode node : array) {
            Question q = new Question();
            q.setKnowledgeId(kp.getId());
            q.setSubjectId(kp.getSubjectId());
            q.setType(node.get("type").asInt());            // 取题型
            q.setDifficulty(node.get("difficulty").asInt());
            q.setContent(node.get("content").asText());
            q.setAnswer(node.get("answer").asText());
            q.setExplanation(node.get("explanation").asText());
            q.setSource(1);                                 // 来源：1=AI 生成

            JsonNode opts = node.get("options");
            if (opts != null && !opts.isNull()) {           // 只有选择题非 null
                q.setOptions(objectMapper.writeValueAsString(opts));  // 选项数组转回 JSON 字符串存库
            }
            questionMapper.insert(q);
            questions.add(q);
        }
    } catch (Exception e) {
        log.error("解析题目失败: {}", e.getMessage());
    }
    return questions;
}
```

要点：

- 提示词把规则写得很死（题型编码、难度、判断题答案约束等），是出题质量的关键。
- `String.format(模板, 值...)`：把知识点标题、说明、数量、题型描述填进模板拼成用户消息。
- `buildTypeDescription` 把 `[1,2,3]` 翻译成"选择题、填空题、判断题"——给模型自然语言比给数字编码更容易理解。
- 选项二次处理：`options` 只有选择题有，非空时用 `writeValueAsString` 把数组转回 JSON 字符串，存进题目的一个字段（没单开表）。
- `source=1` 标明这是 AI 生成的题，便于和人工录入区分。

### 功能 3：流式答疑（最复杂，SSE + 上下文 + 异步）

实现打字机效果：AI 边想边吐字、前端逐字显示。涉及三个技术点：**SSE 推送、对话上下文、异步线程池。**

**第一步：存提问 + 取历史上下文**

```java
public void handleChatStream(SseEmitter emitter, Integer userId, String sessionId, String message, Integer subjectId) {
    // 保存用户本次提问
    ChatHistory userMsg = new ChatHistory();
    userMsg.setUserId(userId);
    userMsg.setSessionId(sessionId);
    userMsg.setSubjectId(subjectId);
    userMsg.setRole("user");
    userMsg.setContent(message);
    chatHistoryMapper.insert(userMsg);

    // 查最近 20 条历史（=10 轮），给 AI 提供上下文
    List<ChatHistory> history = chatHistoryMapper.selectList(
            new LambdaQueryWrapper<ChatHistory>()
                    .eq(ChatHistory::getUserId, userId)
                    .eq(ChatHistory::getSessionId, sessionId)   // 限定同一会话
                    .orderByDesc(ChatHistory::getCreateTime)
                    .last("LIMIT 20"));
    Collections.reverse(history);                           // 倒序查出来，再翻正（旧→新）
```

**为什么带历史？** 大模型无状态，不记得上一句。要连续对话，只能每次把前面的对话原样塞回去。取最近 20 条是在"上下文够用"和"token 不爆"之间折中。`sessionId` 隔离不同对话，不会把别的会话串进来。

**第二步：拼完整消息 + 异步流式调用**

```java
    List<Map<String, String>> fullMessages = new ArrayList<>();
    fullMessages.add(Map.of("role", "system", "content",
            "你是一个耐心的学习辅导老师。请用简洁清晰的语言回答…"));   // 首条系统设定
    for (ChatHistory h : history) {
        fullMessages.add(Map.of("role", h.getRole(), "content", h.getContent()));  // 追加历史
    }

    StringBuilder aiResponse = new StringBuilder();         // 累积 AI 完整回复
    aiExecutor.execute(() -> {                              // 提交到线程池异步执行
        try {
            Map<String, Object> body = new HashMap<>();
            body.put("model", model);
            body.put("messages", fullMessages);
            body.put("max_tokens", maxTokens);
            body.put("temperature", temperature);
            body.put("stream", true);                       // 关键：开启流式

            Request request = new Request.Builder()
                    .url(baseUrl + "/v1/chat/completions")
                    .addHeader("Authorization", "Bearer " + apiKey)
                    .addHeader("Content-Type", "application/json")
                    .post(RequestBody.create(objectMapper.writeValueAsString(body),
                            MediaType.parse("application/json")))
                    .build();

            Response response = httpClient.newCall(request).execute();
            if (!response.isSuccessful()) {
                emitter.send(SseEmitter.event().data("[ERROR] API 调用失败: " + response.code()));
                emitter.complete();
                return;
            }
```

- `stream:true` 让 DeepSeek 不一次性返回，而用 SSE 协议一段段推。
- 用专门的 `aiExecutor` 线程池而非裸 `new Thread`，是为了**控制并发**——答疑慢、并发高时裸线程会无限创建拖垮服务。

**第三步：逐行解析 SSE 流并转推前端**

```java
            try (BufferedReader reader = new BufferedReader(
                    new InputStreamReader(response.body().byteStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {     // 逐行读流式响应
                    if (line.startsWith("data: ")) {             // SSE 数据行以 "data: " 开头
                        String data = line.substring(6).trim();  // 去掉前缀取内容
                        if ("[DONE]".equals(data)) break;        // 结束标志
                        JsonNode chunk = objectMapper.readTree(data);
                        JsonNode delta = chunk.at("/choices/0/delta/content");  // 取增量文本
                        if (delta != null && !delta.isNull()) {
                            String content = delta.asText();
                            aiResponse.append(content);          // 累积到完整回复
                            emitter.send(SseEmitter.event().data(content));  // 实时推送给前端
                        }
                    }
                }
            }
```

- 注意流式取的路径是 `/choices/0/delta/content`（**增量**，每次几个字），和同步取的 `message/content`（完整）不同。
- 每来一小块，一边累积到 `aiResponse`、一边立刻 `emitter.send` 推给浏览器 → 打字机效果。
- `try (BufferedReader ...)` 用 try-with-resources，流用完自动关闭。

**第四步：收尾**

```java
            // 流式结束后，保存 AI 完整回复
            if (aiResponse.length() > 0) {
                ChatHistory aiMsg = new ChatHistory();
                aiMsg.setUserId(userId);
                aiMsg.setSessionId(sessionId);
                aiMsg.setSubjectId(subjectId);
                aiMsg.setRole("assistant");                 // 角色：AI
                aiMsg.setContent(aiResponse.toString());
                chatHistoryMapper.insert(aiMsg);
            }
            emitter.complete();                             // 通知前端推送结束
        } catch (Exception e) {
            log.error("流式对话异常: {}", e.getMessage());
            try {
                emitter.send(SseEmitter.event().data("[ERROR] " + e.getMessage()));
                emitter.complete();
            } catch (IOException ignored) {}                // 连接可能已断，忽略
        }
    });
}
```

把累积的完整回复作为 `assistant` 角色入库，这样**下次对话才能把这轮带进上下文**。整条链路：**提问入库 → 拼历史 → 流式请求 → 增量转推+累积 → 完整回复入库。**

### 功能 4：简答题评分（AI 当阅卷老师）

```java
public Map<String, Object> evaluateAnswer(String questionContent, String correctAnswer, String studentAnswer) {
    // 系统提示词：扮演阅卷老师，按 JSON 返回
    String systemPrompt = """
            你是一个阅卷老师。请根据题目、标准答案和学生的回答，给出评价。
            严格按以下 JSON 格式返回：
            { "isCorrect": true或false, "score": 0到100的整数,
              "feedback": "详细的点评…，50-100字" }
            """;

    String userPrompt = String.format(
            "题目：%s\n标准答案：%s\n学生回答：%s",
            questionContent, correctAnswer, studentAnswer);

    String response = callAi(systemPrompt, userPrompt);

    try {
        String json = response.replaceAll("```json\\s*", "").replaceAll("```\\s*", "").trim();
        return objectMapper.readValue(json, Map.class);     // 解析成 Map 返回
    } catch (Exception e) {
        log.error("解析评价失败: {}", e.getMessage());
        return Map.of("isCorrect", false, "score", 0, "feedback", "评价生成失败，请重试");  // 兜底
    }
}
```

要点：

- 三要素缺一不可：题目（知道考什么）、标准答案（判分依据）、学生答案（被评对象）。
- 返回三字段 JSON：`isCorrect`（对错）、`score`（0–100）、`feedback`（点评）。
- 这是四个功能里**唯一有像样兜底**的——解析失败也返回一个合法结构，前端不会崩（前三个失败只返回空列表/空串）。
- 评分**只返回不入库**（与提取/出题入库不同）。

### 内部方法 1：同步调用 `callAi`（三个功能的发动机）

提取/出题/评分都靠它真正访问大模型。

```java
private String callAi(String systemPrompt, String userPrompt) {
    try {
        List<Map<String, String>> messages = List.of(
                Map.of("role", "system", "content", systemPrompt),
                Map.of("role", "user",   "content", userPrompt));

        Map<String, Object> body = new HashMap<>();
        body.put("model", model);
        body.put("messages", messages);
        body.put("max_tokens", maxTokens);
        body.put("temperature", temperature);
        body.put("stream", false);                          // 同步：一次性拿完整结果

        Request request = new Request.Builder()
                .url(baseUrl + "/v1/chat/completions")      // 目标地址（OpenAI 兼容端点）
                .addHeader("Authorization", "Bearer " + apiKey)  // 鉴权头，没有会 401
                .addHeader("Content-Type", "application/json")
                .post(RequestBody.create(
                        objectMapper.writeValueAsString(body),   // body 转 JSON 字符串
                        MediaType.parse("application/json")))
                .build();

        try (Response response = httpClient.newCall(request).execute()) {  // 发送并等待，用完自动关
            if (!response.isSuccessful()) {                 // 非 2xx
                log.error("AI API 调用失败: {}", response.code());
                return "";
            }
            String responseBody = response.body().string(); // 读响应正文（一整坨 JSON）
            JsonNode root = objectMapper.readTree(responseBody);
            return root.at("/choices/0/message/content").asText("");  // 取 AI 回复正文
        }
    } catch (Exception e) {                                 // 网络/超时/解析等任何异常
        log.error("AI 调用异常: {}", e.getMessage());
        return "";                                          // 一律返回空串
    }
}
```

要点：

- 流程：打包提示词 → 拼成带鉴权的 POST 请求 → 发给 DeepSeek 并等结果 → 从固定路径 `/choices/0/message/content` 抠出回复文本。
- `Authorization: Bearer + apiKey` 是鉴权头，告诉服务器请求是谁、有没有权限，缺了会被拒（401）。
- **失败哲学**：所有失败路径都返回 `""`（鉴权失败、网络异常、超时、解析炸都返回空）。好处是调用方不用层层 try-catch、代码干净；**坏处是把失败原因吞掉了**——上层只看到空串，分不清是 AI 没生成、网络挂了还是密钥过期，只能笼统提示"请重试"。调试时需翻到这里的 `log.error` 才知道真因。

### 内部方法 2：题型编码转描述 `buildTypeDescription`

```java
private String buildTypeDescription(List<Integer> types) {
    Map<Integer, String> typeNames = Map.of(1, "选择题", 2, "填空题", 3, "判断题", 4, "简答题");
    StringBuilder sb = new StringBuilder();
    for (Integer type : types) {
        if (!sb.isEmpty()) sb.append("、");                 // 非首个用顿号分隔
        sb.append(typeNames.getOrDefault(type, "未知题型")); // 编码转名称，未知则兜底
    }
    return sb.toString();
}
```

把 `[1,2,3]` 拼成 `"选择题、填空题、判断题"`，喂给出题提示词时用自然语言而非数字。

## 七、关键设计点与可优化项（重要）

**共性隐患：强依赖大模型严格吐 JSON。** 四个功能（除答疑外的三个）只靠 `replaceAll` 扒 Markdown 这一道防线。一旦模型在 JSON 前后多说话、或字段缺失（如出题漏了 `answer`），`node.get("answer").asText()` 会 NPE 或解析抛异常，被 `catch` 静默吞掉，用户只看到"请重试"。

**改进方向：**

1. **用 API 参数强制 JSON 输出。** DeepSeek 支持 `response_format: {"type":"json_object"}`，让模型从源头只吐 JSON（注意 prompt 里要含 "json" 字样，且它返回的是对象，数组需包一层 `{"items":[...]}` 再取 `items`）。
2. **取字段用 `path()` 而非 `get()`。** `node.path("answer").asText("")` 在字段缺失时返回默认值而不 NPE。
3. **截取法替代 replaceAll。** 从第一个 `[`/`{` 截到最后一个 `]`/`}`，比扒 Markdown 更稳。
4. **解析失败重试一次。** 模型有随机性，重试常能成功。
5. **入库加事务。** 提取和出题是边解析边逐条 insert，中途坏一条前面已入库的不回滚，建议收集完再批量插入并加 `@Transactional`。
6. **评分用更低 temperature。** 评分要稳定可复现，0.7 偏高，调到 0.2 左右更合适（目前四功能共用 0.7）。
7. **`callAi` 失败返回明确原因。** 可改成抛带原因的自定义异常（区分"AI 不可用""返回格式异常"），上层给更准确的提示，也便于排查。

**安全提醒：** `application.yml` 里的 DeepSeek API Key 若以明文随项目泄露，应视为已泄露，需在控制台**立即吊销重置**，今后只用环境变量占位。
