# Java 反射与注解

**反射** 让程序在运行时动态获取类的信息、调用方法。
**注解** 是给代码添加元数据的标记。
两者结合，就是 Spring、MyBatis 等框架的底层基石。

## 一、反射是什么？

**正常情况下：** 编译时就知道用哪个类、调哪个方法
```java
Person p = new Person();
p.setName("小林");
```

**反射情况下：** 运行时才决定用哪个类、调哪个方法
```java
Class<?> clazz = Class.forName("com.example.Person");
Object obj = clazz.getDeclaredConstructor().newInstance();
Method method = clazz.getMethod("setName", String.class);
method.invoke(obj, "小林");
```

> 💡 **反射 = "Java 的上帝模式"**——可以绕过编译期检查，操作任何类。

## 二、获取 Class 对象（反射的入口）

每个类都有一个对应的 `Class` 对象，三种方式获取：

```java
// 方式 1：通过类名.class（最常用）
Class<Person> c1 = Person.class;

// 方式 2：通过对象.getClass()
Person p = new Person();
Class<?> c2 = p.getClass();

// 方式 3：通过全限定类名（动态加载）
Class<?> c3 = Class.forName("com.example.Person");

// 三者得到的是同一个 Class 对象
System.out.println(c1 == c2);   // true
```

## 三、通过反射获取类信息

```java
Class<?> clazz = Person.class;

// 类名
clazz.getName();                // com.example.Person（全限定名）
clazz.getSimpleName();          // Person（简短名）

// 父类
clazz.getSuperclass();          // class java.lang.Object

// 实现的接口
clazz.getInterfaces();

// 修饰符（public、abstract 等）
int mod = clazz.getModifiers();
Modifier.isPublic(mod);         // true
```

## 四、操作字段（Field）

```java
public class Person {
    public String name;
    private int age;
}
```

```java
Class<?> clazz = Person.class;
Person p = new Person();

// 获取所有公开字段
Field[] fields = clazz.getFields();          // 只能获取 public 字段
Field[] all = clazz.getDeclaredFields();     // 所有字段（包括 private）

// 获取指定字段
Field nameField = clazz.getField("name");
Field ageField = clazz.getDeclaredField("age");

// 读取字段值
String name = (String) nameField.get(p);

// 修改字段值
nameField.set(p, "小林");

// 访问 private 字段需要先"解锁"
ageField.setAccessible(true);
ageField.set(p, 18);
System.out.println(ageField.get(p));   // 18
```

## 五、操作方法（Method）

```java
public class Person {
    public void sayHello() {
        System.out.println("Hello");
    }

    private int add(int a, int b) {
        return a + b;
    }
}
```

```java
Class<?> clazz = Person.class;
Person p = new Person();

// 获取方法
Method m1 = clazz.getMethod("sayHello");                          // 公开方法
Method m2 = clazz.getDeclaredMethod("add", int.class, int.class); // 含参数类型

// 调用方法
m1.invoke(p);                            // 等价于 p.sayHello()

// 调用 private 方法
m2.setAccessible(true);
int result = (int) m2.invoke(p, 3, 5);   // 8
```

## 六、操作构造方法

```java
public class Person {
    public Person() { }
    public Person(String name) { this.name = name; }
    private Person(int age) { this.age = age; }

    public String name;
    public int age;
}
```

```java
Class<?> clazz = Person.class;

// 无参构造
Constructor<?> c1 = clazz.getDeclaredConstructor();
Person p1 = (Person) c1.newInstance();

// 有参构造
Constructor<?> c2 = clazz.getDeclaredConstructor(String.class);
Person p2 = (Person) c2.newInstance("小林");

// private 构造
Constructor<?> c3 = clazz.getDeclaredConstructor(int.class);
c3.setAccessible(true);
Person p3 = (Person) c3.newInstance(18);
```

## 七、反射的实际用途

### 1. 框架根据配置动态创建对象

```java
// Spring 的核心逻辑就是这样：
String className = config.get("bean.class");   // 从配置文件读类名
Class<?> clazz = Class.forName(className);
Object bean = clazz.getDeclaredConstructor().newInstance();
```

### 2. 通用工具方法

```java
// 把 Map 转成对象
public static <T> T mapToObject(Map<String, Object> map, Class<T> clazz) throws Exception {
    T obj = clazz.getDeclaredConstructor().newInstance();
    for (Map.Entry<String, Object> entry : map.entrySet()) {
        Field field = clazz.getDeclaredField(entry.getKey());
        field.setAccessible(true);
        field.set(obj, entry.getValue());
    }
    return obj;
}
```

### 3. 单元测试访问私有方法

```java
// JUnit 测试 private 方法时常用
Method m = clazz.getDeclaredMethod("privateMethod");
m.setAccessible(true);
m.invoke(obj);
```

## 八、反射的优缺点

✅ **优点：**
- 灵活，运行时动态操作类
- 框架开发的基础

❌ **缺点：**
- **性能差**（比直接调用慢几十倍，但对一次性操作影响不大）
- **破坏封装**（可以访问 private 成员）
- 编译器无法检查，错误推迟到运行时
- 代码可读性差

> 💡 **业务代码慎用反射，框架代码大量使用反射。**

# 注解（Annotation）

## 一、注解是什么？

**注解 = 给代码贴的"标签"**，本身不影响代码逻辑，但可以被工具/框架读取并做出反应。

最常见的内置注解：
```java
@Override               // 表示重写父类方法
@Deprecated             // 表示该方法已过时
@SuppressWarnings("xxx") // 抑制编译警告
```

## 二、定义自己的注解

```java
import java.lang.annotation.*;

@Target(ElementType.METHOD)         // 注解可用在哪里
@Retention(RetentionPolicy.RUNTIME) // 注解保留到什么时候
public @interface MyAnnotation {
    String value() default "默认值";   // 注解的属性
    int priority() default 0;
}
```

### @Target —— 注解能贴在哪

| 取值                     | 说明           |
| ---------------------- | ------------ |
| `ElementType.TYPE`     | 类、接口、枚举      |
| `ElementType.FIELD`    | 字段           |
| `ElementType.METHOD`   | 方法           |
| `ElementType.PARAMETER`| 方法参数         |
| `ElementType.CONSTRUCTOR` | 构造方法     |
| `ElementType.LOCAL_VARIABLE` | 局部变量   |

### @Retention —— 注解保留多久

| 取值                          | 说明                       |
| --------------------------- | ------------------------ |
| `RetentionPolicy.SOURCE`    | 只在源码，编译后丢弃（如 `@Override`）|
| `RetentionPolicy.CLASS`     | 编译到字节码，运行时不可见（默认）        |
| `RetentionPolicy.RUNTIME`   | 运行时可读（**反射用这个**）         |

## 三、使用自定义注解

```java
public class Service {
    @MyAnnotation(value = "Hello", priority = 1)
    public void doSomething() {
        System.out.println("干活");
    }

    @MyAnnotation
    public void doOther() { }
}
```

## 四、通过反射读取注解

```java
Class<?> clazz = Service.class;

for (Method method : clazz.getDeclaredMethods()) {
    // 判断是否有这个注解
    if (method.isAnnotationPresent(MyAnnotation.class)) {
        // 获取注解
        MyAnnotation anno = method.getAnnotation(MyAnnotation.class);
        System.out.println(method.getName() + " → "
            + anno.value() + ", priority=" + anno.priority());
    }
}

// 输出：
// doSomething → Hello, priority=1
// doOther → 默认值, priority=0
```

## 五、综合实战：手写一个迷你测试框架

模拟 JUnit 的 `@Test` 注解：

```java
// 1. 定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyTest { }

// 2. 写测试类
public class CalculatorTest {
    @MyTest
    public void testAdd() {
        if (1 + 1 != 2) throw new RuntimeException("失败");
        System.out.println("testAdd 通过");
    }

    @MyTest
    public void testSub() {
        if (5 - 3 != 2) throw new RuntimeException("失败");
        System.out.println("testSub 通过");
    }

    public void notATest() {
        System.out.println("这不是测试方法");
    }
}

// 3. 写测试运行器（用反射 + 注解）
public class TestRunner {
    public static void run(Class<?> clazz) throws Exception {
        Object instance = clazz.getDeclaredConstructor().newInstance();
        for (Method method : clazz.getDeclaredMethods()) {
            if (method.isAnnotationPresent(MyTest.class)) {
                try {
                    method.invoke(instance);
                } catch (Exception e) {
                    System.out.println(method.getName() + " 失败：" + e.getCause().getMessage());
                }
            }
        }
    }

    public static void main(String[] args) throws Exception {
        run(CalculatorTest.class);
    }
}

// 输出：
// testAdd 通过
// testSub 通过
// （notATest 没有 @MyTest 注解，被跳过）
```

> 🤯 **JUnit 就是这么工作的。** Spring 的 `@Autowired`、`@Controller` 也是同样原理——**注解 + 反射** 就是框架"魔法"的真相。

## 六、Spring 中你将看到的注解（提前感受）

```java
@RestController                     // 标记这是一个控制器
@RequestMapping("/api/users")
public class UserController {

    @Autowired                       // 自动注入 Bean
    private UserService userService;

    @GetMapping("/{id}")             // GET 请求路由
    public User getUser(@PathVariable Long id) {
        return userService.findById(id);
    }

    @PostMapping                     // POST 请求路由
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
}
```

**它们怎么生效的？**
1. Spring 启动时**扫描所有类**
2. 用反射读取每个类上的注解
3. 根据注解类型做相应处理（注册 Bean、绑定路由等）

学完反射和注解，这些"黑魔法"对你就不再神秘了。

## 七、最佳实践

✅ **该做的：**
- 框架/工具类开发可以放心用反射
- 自定义注解要设置合理的 `@Target` 和 `@Retention`
- 用注解替代 XML 配置，代码更直观

❌ **不该做的：**
- 业务代码不要滥用反射（可读性差、性能差）
- 不要用反射修改 `final` 字段（行为不确定）
- 不要忽略 `setAccessible(true)` 的安全风险
